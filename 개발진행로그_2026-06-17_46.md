# 개발 진행 로그 46

## 목표

- 전체 백업 복원 시 업로드 배치 메타데이터뿐 아니라 DB에 보관된 원본 CSV 텍스트까지 복원한다.

## 작업 내용

- 워크스페이스 백업 payload에 `originalImportFiles` 배열을 추가했다.
- `백업 JSON` 다운로드 시 보관된 원본 CSV를 조회해 JSON 안에 포함한다.
- `백업 ZIP` 다운로드 시 동일한 원본 CSV 데이터를 `workspace-backup.json`과 `imports/original-csv/` 파일 양쪽에 포함한다.
- 복원 API `POST /api/backups/restore`가 `originalImportFiles`를 받아 `ImportBatch.originalFileText`로 복원하도록 했다.
- dry-run 응답의 `restoredCounts`에 `originalImportFiles` 개수를 추가했다.
- `verify:backup-restore` 검증 스크립트가 원본 CSV 포함 백업을 dry-run으로 검사하도록 확장했다.
- README의 MVP 범위를 원본 CSV 포함 전체 백업/복원으로 갱신했다.

## 복원 대상 추가

- `ImportBatch.originalFileText`
- `ImportBatch.originalFileMimeType`
- `ImportBatch.originalFileSize`
- `ImportBatch.originalFileHash`
- `ImportBatch.originalFileName`
