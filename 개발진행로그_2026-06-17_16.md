# 개발진행로그 16

## 목표

증빙 입력 시 사용자가 거래를 직접 찾는 부담을 줄이고, 비용 증빙 매칭 누락을 줄인다.

## 변경 내용

- 증빙 추가 화면에 `추천 매칭` 영역을 추가했다.
- 합계 금액, 거래처, 발행일을 기준으로 거래 후보를 점수화한다.
- 금액이 일치하면 가장 높은 우선순위로 표시하고, 거래처 유사도와 일자 근접성을 보조 점수로 사용한다.
- 이미 매칭된 거래는 점수를 낮춰 새 증빙 후보에서 뒤로 밀리게 했다.
- 추천 후보를 클릭하면 `거래 매칭` 선택값이 자동으로 설정된다.

## 검증 결과

- `npm run lint` 통과
- `npm run typecheck` 통과
- `npm run verify:samples` 통과
- `npm run build` 통과
