# 개발 진행 로그 - 2026-06-18 29

## 목표
- CSV 매핑이 올바르더라도 실제 행 값이 비어 있거나 잘못된 경우 서버에서 거래 생성을 차단한다.
- 날짜 파싱 실패 시 오늘 날짜로 대체되거나, 0원 거래가 저장되는 경로를 막는다.

## 변경 내용
- `src/app/api/imports/route.ts`
  - `validateImportRows`를 추가했다.
  - 각 행의 거래일, 내용/적요, 실제 금액 값을 검증한다.
  - 거래일은 엄격한 날짜 검증을 통과해야 하며, 실패 시 `INVALID_CSV_ROWS`로 거부한다.
  - 금액/입금/출금 값이 모두 0 이하이거나 입금과 출금이 동시에 있는 행을 거부한다.
  - 행 오류는 최대 25개까지만 반환하고 초과분은 요약한다.
- `scripts/smoke-production.mjs`
  - 매핑은 유효하지만 날짜/내용/금액 값이 잘못된 CSV import 요청이 400으로 차단되는지 확인한다.

## 검증
- `npm run typecheck` 통과.
- `npm run lint` 통과.
- 외부 실행 `npm run verify:mvp` 통과.

## 판단
- CSV 업로드 API가 단순 매핑 검증을 넘어 행 값 품질까지 서버에서 방어한다.
- 신고 준비 데이터의 기본 전제인 거래일, 적요, 금액이 빈 값 또는 잘못된 값으로 저장될 가능성을 줄였다.
