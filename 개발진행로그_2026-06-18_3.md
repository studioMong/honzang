# 개발진행로그 2026-06-18 3

## 작업 범위
- 자동분개 생성 로직에서 외주비와 부가세 처리 기준을 보정했다.
- 복식부기를 모르는 사용자가 외주비 세금계산서 거래를 승인할 수 있도록 차변/대변 균형 초안을 만들게 했다.

## 변경 내용
- `src/lib/accounting.ts`
  - 분개 생성 시 총액, 공급가액, 부가세 계산 순서를 명확히 했다.
  - 부가세 정보가 명시된 외주비(`WITHHOLDING_REVIEW`)는 `외주비 + 부가세대급금 / 보통예금` 구조로 분개한다.
  - 부가세 정보가 없는 외주비 출금은 총액을 비용으로 처리해 초안이 불균형이 되지 않도록 했다.
  - 해외 SaaS 신호가 있는 지출은 매입부가세로 자동 공제하지 않고 총액 비용 처리한다.
- `scripts/verify-sample-csvs.ts`
  - 외주비 총액 비용 처리 케이스와 외주비 세금계산서 케이스의 분개 균형 검증을 추가했다.

## 검증
- `npm run typecheck` 통과.
- `npm run lint` 통과.
- `npm run verify:samples` 통과.
- `npm run build` 통과.
- `npm run verify:mvp` 통과.
- 브라우저 확인:
  - 자동분개 탭에서 `프리랜서 디자인 외주비` 초안이 `502 외주비 ₩300,000`, `135 부가세대급금 ₩30,000`, `101 보통예금 ₩330,000`으로 표시되는 것을 확인했다.
  - 해당 초안의 `승인 저장` 버튼이 활성화되어 차변/대변 불균형으로 막히지 않는 것을 확인했다.
  - 데스크톱 1280px에서 문서 가로 넘침 없음 확인.

## 남은 판단
- 실제 외주비가 세금계산서 거래인지 3.3% 원천세 거래인지는 사용자가 확인해야 하므로 검토 경고는 유지한다.
- 향후에는 거래처 기본값의 원천세 유형에 따라 `예수금` 라인을 제안하는 보조 흐름을 추가할 수 있다.
