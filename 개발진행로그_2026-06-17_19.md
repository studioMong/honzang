# 개발진행로그 19

## 목표

Railway Postgres 연결 후 빈 DB나 추가 CSV 업로드 상황에서 화면 상태가 실제 장부와 다르게 보이는 문제를 제거한다.

## 변경 내용

- DB 연결 모드에서는 거래/증빙 API가 빈 배열을 반환해도 샘플 데이터로 대체하지 않도록 수정했다.
- 샘플 데이터 fallback은 API들이 모두 sample mode일 때만 사용한다.
- CSV import 후 새 배치 거래만 화면에 남기던 동작을 기존 거래와 id 기준 병합하도록 수정했다.
- 거래 병합 결과는 거래일 최신순으로 정렬한다.

## 수정 이유

- 실제 Postgres DB가 비어 있는 초기 상태에서 샘플 거래가 보이면 사용자가 실제 데이터와 데모 데이터를 혼동할 수 있다.
- 여러 자료를 순차적으로 업로드할 때 이전 거래가 화면에서 사라지면 신고 준비 리포트가 일시적으로 잘못 계산된다.
