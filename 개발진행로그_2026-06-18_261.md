## 목표

- Railway 프로덕션 배포에서 `HONZANG_ACCESS_CODE`만 설정하고 `HONZANG_ACCESS_TOKEN_SALT`를 누락한 경우, 접근 쿠키가 발급되지 않는지 자동 검증한다.
- 새 운영 guard를 전체 MVP 검증 흐름과 README 실행 문서에 포함한다.

## 변경 사항

- `scripts/verify-access-salt.mjs`를 추가했다.
  - `.next/standalone/server.js`가 없는 경우 빌드 선행 안내와 함께 실패한다.
  - `NODE_ENV=production`, `HONZANG_ACCESS_CODE` 설정, `HONZANG_ACCESS_TOKEN_SALT` 미설정 상태로 standalone 서버를 실행한다.
  - `/api/auth/session`이 공개 진단으로 응답하되 인증 상태는 false인지 확인한다.
  - 올바른 접근 코드를 제출해도 HTTP 500으로 fail closed 처리되고, `set-cookie`가 비어 있는지 확인한다.
- `package.json`에 `verify:access-salt` 스크립트를 추가했다.
- `scripts/verify-mvp.mjs` 전체 검증 순서에 `verify:access-salt`를 추가했다.
- `README.md`의 PWA/접근 제어 검증 절차에 `npm run verify:access-salt`와 목적 설명을 추가했다.

## 검증

## 배포 메모

- Railway 환경변수에는 최소 `DATABASE_URL`, `NEXT_PUBLIC_APP_URL`, `HONZANG_ACCESS_CODE`, `HONZANG_ACCESS_TOKEN_SALT`, `HONZANG_FILE_ENCRYPTION_KEY`가 필요하다.
- 이번 guard는 `HONZANG_ACCESS_TOKEN_SALT` 누락 상태에서 접근코드가 맞아도 쿠키를 만들지 않는 동작을 보장한다.
