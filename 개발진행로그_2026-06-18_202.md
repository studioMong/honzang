# 개발 진행 로그 202

## 작업 주제
- 회당 정산/SaaS 과금 단위 추정 로직을 독립 검증 가능한 라이브러리로 분리.

## 변경 사항
- `src/lib/billing.ts`를 추가했다.
  - 과금 모델 라벨, 활성 단가 선택, 매출 거래 판정, 공급가액 추정, 과금 단위 계산, 단위 포맷을 한 곳에서 관리한다.
  - 내부 회당 정산은 `perUseUnitPrice`, SaaS 월 구독은 `monthlySubscriptionPrice`, SaaS 연 구독은 `annualSubscriptionPrice`를 기준 단가로 사용한다.
  - 명시된 매출 계정은 매출로 포함하고, 명시된 비매출 계정은 `구독`/`정산` 문구가 있어도 과금 매출에서 제외한다.
- `src/components/app-workspace.tsx`의 과금 정책 카드와 설정 준비도 로직이 새 `billing` 라이브러리를 사용하도록 정리했다.
- `scripts/verify-billing-estimates.ts`를 추가했다.
  - 세 가지 과금 모델의 단가 선택과 단위 라벨을 검증한다.
  - 계정과목 기반 매출 판정, 홈택스 매출, 구독/정산 문구 기반 매출 판정, 출금 제외, 비매출 계정 우선순위를 검증한다.
  - 공급가액 직접 입력과 부가세 포함 입금액 기준 추정, 0원 단가 방어, 과금 단위 포맷을 검증한다.
- `package.json`에 `verify:billing-estimates`를 추가했다.
- `verify:mvp`에 `verify:billing-estimates` 단계를 추가했다.

## 검증
- `npm run typecheck` 통과.
- `npm run verify:billing-estimates` 통과.
  - 샌드박스에서는 `tsx` IPC pipe 제한으로 실패했고, 권한 상승 후 동일 명령은 통과했다.
- `npm run lint` 통과.
- `git diff --check` 통과.
- `npm run verify:mvp` 통과.
