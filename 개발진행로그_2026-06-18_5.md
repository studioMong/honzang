# 개발진행로그 2026-06-18 5

## 작업 범위
- 거래처 기본값의 원천세 유형을 자동분개 초안에 반영했다.
- `사업소득 3.3%`로 설정된 외주 거래는 원천세 예수금 라인을 추정해 차변/대변이 맞는 초안을 만들도록 했다.

## 변경 내용
- `src/lib/accounting.ts`
  - 거래처 원천세 유형 경고 문구를 내부 코드가 아니라 사용자 친화 문구로 저장하도록 변경했다.
  - `사업소득 3.3%` 신호가 있는 외주비 거래는 순지급액을 역산해 `외주비 / 보통예금 + 예수금` 분개를 생성한다.
  - 기타소득, 급여는 계산 기준이 다양하므로 자동 예수금 분개 대신 검토 경고로 남긴다.
- `scripts/verify-sample-csvs.ts`
  - 거래처 기본값 `BUSINESS_INCOME` 적용 시 외주비 총액과 예수금이 균형을 이루는지 검증하는 케이스를 추가했다.

## 검증
- `npm run typecheck` 통과.
- `npm run lint` 통과.
- `npm run verify:samples` 통과.
- `npm run verify:mvp` 통과.

## 판단
- 1인법인이 외주비를 지급할 때 세금계산서 거래와 3.3% 원천세 거래를 장부상 다르게 처리할 수 있는 기반이 생겼다.
- 최종 신고 전에는 실제 계약 형태, 지급명세서 대상 여부, 원천세 신고 대상 여부를 사용자가 확인해야 하므로 검토 경고는 유지한다.
