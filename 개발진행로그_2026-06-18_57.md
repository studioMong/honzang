# 개발 진행 로그 2026-06-18 57

## 작업 범위
- 거래 금액 데이터가 입금/출금을 동시에 갖거나 방향과 금액이 충돌하지 않도록 서버 검증을 보강했다.

## 구현 내용
- `src/lib/server/transaction-validation.ts`
  - `validateTransactionAmounts` 공통 헬퍼를 추가했다.
  - 입금/출금 금액이 모두 0 이하인 거래를 거절한다.
  - 입금/출금 금액이 동시에 0보다 큰 거래를 거절한다.
  - `DEPOSIT` 방향은 입금 금액, `WITHDRAWAL` 방향은 출금 금액이 0보다 커야 한다.
- `src/app/api/transactions/route.ts`
  - 수동 거래 생성 시 공통 금액 검증을 사용한다.
  - 금액 오류는 `INVALID_TRANSACTION_AMOUNTS` 코드와 메시지로 반환한다.
- `src/app/api/backups/restore/route.ts`
  - 백업 복원 dry-run에서 거래 금액 모순을 `INVALID_BACKUP_TRANSACTIONS`로 거절한다.

## 테스트 보강
- `scripts/smoke-production.mjs`
  - 수동 거래 생성 API가 입금/출금 동시 입력을 HTTP 400과 `INVALID_TRANSACTION_AMOUNTS`로 거절하는지 확인한다.
- `scripts/verify-backup-restore.mjs`
  - 백업 JSON 안의 거래 금액 모순을 `INVALID_BACKUP_TRANSACTIONS`로 거절하는지 확인한다.
