# 개발 진행 로그 89

## 작업 단위
- 반복되는 Railway 공개 도메인 오연결 문제를 해결하기 위한 배포 전환 체크리스트를 문서화했다.

## 변경 내용
- `docs/railway-cutover.md`
  - 목표 배포 상태를 명시했다.
  - Railway Dashboard에서 확인해야 하는 GitHub 연결, Dockerfile builder, deploy 명령, healthcheck, Variables, public domain 연결 순서를 정리했다.
  - 로컬 MVP 검증, staging DB workflow 검증, 운영 URL 감사 명령을 정리했다.
  - `/api/version`, `/api/health`, `/manifest.webmanifest`가 404인 반복 실패 패턴을 legacy/static 서비스 연결 문제로 분류했다.
- `README.md`
  - Railway 공개 URL이 static/legacy 서비스에 연결된 경우 `docs/railway-cutover.md`를 따르도록 링크를 추가했다.
- `scripts/verify-deployment-config.mjs`
  - README가 Railway 전환 체크리스트를 링크하는지 확인한다.
  - 체크리스트에 production domain, `/api/version`, `/api/health`, `/manifest.webmanifest`, `VERIFY_DB_WORKFLOW_BASE_URL` 안내가 포함되어 있는지 검증한다.

## 검증
- `npm run verify:deployment-config` 통과
- `npm run typecheck` 통과
- `npm run lint` 통과
- `git diff --check` 통과
- `npm run verify:mvp` 통과
