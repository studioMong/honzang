# 개발 진행 로그 117

## 작업 단위
- 백업 JSON/ZIP 내보내기 후 원본 CSV와 증빙 파일 포함 상태를 사용자에게 명확히 보여주도록 보강했다.

## 변경 파일
- `src/components/app-workspace.tsx`

## 구현 내용
- 백업 JSON/ZIP 생성 전 기존 백업 메시지를 초기화하고, 생성 완료 후 포함 상태 메시지를 표시한다.
- 원본 CSV 포함 수, 원본 CSV 누락 수, DB 보관 증빙 수, 외부 링크 증빙 수, 파일 없는 증빙 수를 백업 메시지에 표시한다.
- 백업 생성 실패 시 JSON/ZIP별 실패 메시지를 표시한다.
- ZIP `manifest.json`에 `originalCsvFileSummary`와 `evidenceFileSummary`를 추가해, 압축 파일 안에서도 누락/외부 링크 상태를 확인할 수 있게 했다.

## 실행 명령
- `npm run typecheck`
- `npm run lint`
- `git diff --check`
- `npm run verify:mvp`
- `if [ -n "$DATABASE_URL" ]; then printf 'DATABASE_URL=set\n'; else printf 'DATABASE_URL=missing\n'; fi`
