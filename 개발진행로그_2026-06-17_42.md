# 개발 진행 로그 42

## 목표

- Railway DB 장애, 서비스 이전, 수동 검토에 대비해 전체 워크스페이스 데이터를 파일로 백업할 수 있게 만든다.

## 작업 내용

- 설정 화면에 `전체 백업` 섹션을 추가했다.
- `백업 JSON` 버튼을 추가해 현재 앱 상태 전체를 JSON으로 내려받을 수 있게 했다.
- `백업 ZIP` 버튼을 추가해 구조화 JSON과 가능한 원본 파일을 함께 묶어 내려받을 수 있게 했다.
- 백업 JSON에는 다음 데이터를 포함한다.
  - 회사 정보
  - 계정과목
  - CSV 템플릿
  - 업로드 배치 메타데이터
  - 거래
  - 증빙
  - 분개
  - 저장 리포트
  - 거래처 기본값
  - 자동 분류 규칙
  - 검토 항목
- ZIP 백업에는 `workspace-backup.json`, 가능한 원본 CSV 파일, DB 보관 증빙 파일을 포함한다.
- ZIP 백업 `manifest.json`에 데이터 개수, 원본 CSV 파일 경로, 증빙 파일 경로를 기록한다.
- README의 MVP 범위에 전체 워크스페이스 백업 JSON/ZIP 다운로드를 추가했다.

## 검증

- `npm run typecheck` 통과
- `npm run lint` 통과
- `npm run build` 통과
- `npm run verify:samples` 통과
- `npm run smoke:prod` 통과
- `npm run verify:pwa` 통과
- 로컬 production 서버 설정 화면 SSR 확인
  - `/?view=settings` 200 응답
  - `전체 백업` 문구 확인
  - `백업 JSON` 문구 확인
  - `백업 ZIP` 문구 확인

## 비고

- 이번 작업은 내보내기 백업이다. 자동 복원 기능은 아직 추가하지 않았다.
- ZIP 백업은 원본 CSV 단건 조회가 실패해도 구조화 JSON과 나머지 파일 백업은 계속 생성되도록 했다.
