# 개발 진행 로그 - 2026-06-17 4차

## 이번 목표
- 자동분개 초안을 실제 분개 데이터로 승인 저장할 수 있게 만든다.
- 승인된 분개를 바탕으로 계정별 원장을 생성하고 CSV로 내보낼 수 있게 만든다.

## 구현 내용
- `/api/journals` 라우트를 추가했다.
  - `GET`: 회사의 분개 목록을 조회한다.
  - `POST`: 초안 라인의 차변/대변 균형을 검증한 뒤 승인 분개와 분개 라인을 저장한다.
  - 데이터베이스가 없을 때는 샘플 모드 응답으로 UI 흐름을 유지한다.
- 자동분개 화면에 `승인 저장` 동작을 추가했다.
  - 승인 성공 시 해당 거래는 `승인됨`, 버튼은 `저장 완료` 상태로 표시된다.
  - 이미 승인된 거래는 같은 화면에서 중복 저장하지 않도록 막았다.
- 리포트 화면에 `계정별 원장` 섹션을 추가했다.
  - 승인된 분개 라인을 계정별/일자별로 정렬한다.
  - 자산/비용은 차변 증가, 부채/자본/수익은 대변 증가 기준으로 계정별 누적 잔액을 계산한다.
  - `honzang-ledger.csv` 내보내기를 제공한다.
- 공통 타입과 Prisma 직렬화 계층에 `AppJournalEntry`를 추가했다.

## 검증
- `npm run lint` 통과.
- `npm run typecheck` 통과.
- `npm run build` 통과.
- 로컬 브라우저 확인:
  - `/?view=journals`에서 `자동분개 초안`, `승인 저장`, `균형` 표시 확인.
  - 첫 번째 샘플 분개 승인 후 `승인됨`, `저장 완료` 표시 확인.
  - `/?view=reports`에서 승인 분개 1건 기준 계정별 원장 행 생성 확인.
- 로컬 API 확인:
  - `GET /api/journals`가 샘플 모드에서 `{ "journalEntries": [], "mode": "sample" }` 형태로 응답하는 것을 확인했다.
- Railway 배포 URL 확인:
  - `https://honzang-production.up.railway.app`는 아직 새 Next 앱이 아니라 기존 `혼자장부 프로젝트 정리` 정적 HTML을 서빙한다.
  - 응답 헤더는 `server: railway-hikari`, `content-length: 24489`로 확인됐다.
