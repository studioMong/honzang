# 개발 진행 로그 214

## 작업 주제
- 월 마감 잠금 정책 검증을 CSV 가져오기/업로드 삭제까지 확장했다.

## 구현 내용
- `scripts/verify-period-locks.mjs`를 추가해 마감 잠금 계약을 정적으로 검증한다.
- 검증 대상은 다음이다.
  - CSV 가져오기 생성/삭제가 `findClosedPeriodForDates`와 `PERIOD_CLOSED` 응답을 사용하는지 확인
  - 수기 거래 생성/수정이 마감 기간을 확인하는지 확인
  - 증빙 생성/삭제가 증빙 발행일과 연결 거래일 양쪽의 마감 기간을 확인하는지 확인
  - 분개 생성/상태 변경이 분개일과 연결 거래일 양쪽의 마감 기간을 확인하는지 확인
  - 리포트와 검토함 변경이 마감 기간을 확인하는지 확인
- `scripts/verify-db-workflow.ts`를 보강해 실제 DB 검증 환경에서는 마감 후 CSV 재가져오기와 업로드 삭제가 `409 PERIOD_CLOSED`로 막히는지 확인하도록 했다.
- `npm run verify:period-locks`를 추가하고 `npm run verify:mvp`에 연결했다.
- README MVP 범위에 마감된 기간의 CSV 가져오기와 업로드 삭제 차단을 명시했다.

## 실행한 검증
- `npm run typecheck`: 통과
- `npm run verify:period-locks`: 통과
- `npm run lint`: 통과
- `npm run verify:deployment-config`: 통과
- `git diff --check`: 통과
- `npm run verify:mvp`: 통과

## 미실행 검증
- `npm run verify:db-workflow`: 현재 셸에 `DATABASE_URL`이 없어 실행하지 않았다.
- 단, DB workflow 스크립트에는 잠긴 기간 CSV 가져오기/업로드 삭제 차단 검증을 추가해 DB 연결 환경에서 실행되도록 준비했다.
