# 개발 진행 로그 127

## 작업 단위
- CSV 업로드 생성 경로를 원자 작업으로 묶어 중간 실패 시 빈 업로드 이력이나 부분 저장 데이터가 남지 않도록 보강했다.

## 변경 파일
- `src/app/api/imports/route.ts`
- `scripts/verify-db-workflow.ts`

## 구현 내용
- 신규 CSV 업로드에서 `ImportBatch` 생성, CSV 템플릿 저장, 거래 생성, 감사 로그 기록을 하나의 Prisma 트랜잭션으로 묶었다.
- 중복 CSV 업로드에서도 CSV 템플릿 저장과 기존 배치 원본 파일 보강을 하나의 트랜잭션으로 처리한다.
- DB 워크플로우 검증에 동일 CSV 재업로드 시 `duplicate: true`가 반환되고 기존 `importBatchId`와 거래 목록을 재사용하는 시나리오를 추가했다.

## 실행 명령
- `npm run typecheck`
- `npm run lint`
- `git diff --check`
- `npm run verify:mvp`
- `if [ -n "$DATABASE_URL" ]; then printf 'DATABASE_URL=set\n'; else printf 'DATABASE_URL=missing\n'; fi`
