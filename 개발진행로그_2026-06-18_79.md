# 개발 진행 로그 79

## 작업 단위
- 거래의 공급가액/부가세 분해 금액이 실제 입금/출금 총액을 초과하지 못하도록 서버 검증을 추가했다.

## 변경 내용
- `src/lib/server/transaction-validation.ts`
  - 거래 금액 검증에 공급가액/부가세 분해 검증을 통합했다.
  - 공급가액이 거래 총액보다 큰 경우, 부가세가 거래 총액보다 큰 경우, 공급가액+부가세 합계가 거래 총액보다 큰 경우를 차단한다.
- `src/app/api/imports/route.ts`
  - CSV 가져오기 행 검증에서 매핑된 공급가액/부가세를 읽어 공통 거래 세액 검증을 적용했다.
- `scripts/smoke-production.mjs`
  - 수기 거래와 CSV 가져오기에서 세액 분해 금액이 총액을 초과하는 실패 케이스를 추가했다.
- `scripts/verify-backup-restore.mjs`
  - 백업 복원 dry-run에서 잘못된 거래 세액 분해 데이터를 거절하는 회귀 검증을 추가했다.

## 검증
- `npm run typecheck` 통과
- `npm run lint` 통과
- `git diff --check` 통과
- `npm run verify:mvp` 통과
