# 개발 진행 로그 93

## 작업 단위
- 운영 DB 전체 교체 성격의 백업 복원 API에 명시 확인 문구 가드를 추가했다.

## 변경 내용
- `src/lib/backup-restore.ts`
  - 서버와 클라이언트가 공유하는 복원 확인 문구 `혼자장부 전체교체`를 추가했다.
- `src/app/api/backups/restore/route.ts`
  - 실제 복원 요청은 `confirmReplace: true`와 `restoreConfirmation: "혼자장부 전체교체"`가 모두 맞을 때만 통과한다.
  - 확인값이 없거나 틀리면 `RESTORE_CONFIRMATION_REQUIRED` 코드로 400 응답을 반환한다.
- `src/components/app-workspace.tsx`
  - 드라이런 결과 확인 후 사용자가 확인 문구를 직접 입력해야 복원 요청을 보낸다.
  - 문구가 일치하지 않으면 복원을 취소하고 안내 메시지를 표시한다.
- `scripts/verify-backup-restore.mjs`
  - `confirmReplace` 누락, 잘못된 확인 문구, 올바른 확인 문구 후 sample-mode DB 가드 도달을 검증한다.

## 실행 명령
- `npm run typecheck`
- `npm run lint`
- `git diff --check`
- `npm run verify:mvp`

## 검증 결과
- TypeScript typecheck 통과.
- ESLint 통과.
- Git whitespace 검사 통과.
- MVP verification passed.
- `verify:backup-restore`가 새 복원 확인 문구 가드를 통과했다.

## 판단
- 백업 파일 선택과 기본 확인창만으로 전체 DB 교체 요청이 나가지 않도록 API와 UI 양쪽에서 안전장치를 강화했다.
- Railway 배포 후 Postgres 운영 데이터 복원 작업의 실수 위험이 줄었다.
