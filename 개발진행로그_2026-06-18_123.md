# 개발 진행 로그 123

## 작업 단위
- 자동 분류 규칙 수정/삭제 시 존재하지 않는 규칙 ID가 Prisma 예외로 이어지지 않도록 404 가드를 보강했다.

## 변경 파일
- `src/app/api/classification-rules/route.ts`
- `scripts/verify-db-workflow.ts`

## 구현 내용
- `PATCH /api/classification-rules`에서 회사 소유 규칙을 먼저 조회하고, 없으면 HTTP 404와 `자동 분류 규칙을 찾을 수 없습니다.` 메시지를 반환한다.
- `DELETE /api/classification-rules`도 삭제 전 존재 여부를 확인하고, 없으면 동일하게 404를 반환한다.
- 기존 계정과목 존재 검증은 조회된 규칙이 있을 때만 진행하도록 정리했다.
- DB 워크플로우 검증에 없는 규칙 ID 수정/삭제가 404로 응답하는 시나리오를 추가했다.

## 실행 명령
- `npm run typecheck`
- `npm run lint`
- `git diff --check`
- `npm run verify:mvp`
- `if [ -n "$DATABASE_URL" ]; then printf 'DATABASE_URL=set\n'; else printf 'DATABASE_URL=missing\n'; fi`
