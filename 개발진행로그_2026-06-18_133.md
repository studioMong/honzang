# 개발 진행 로그 133

## 작업 단위
- 이미 마감 잠금된 월의 스냅샷이 API 직접 호출로 덮어써지는 것을 차단했다.

## 변경 파일
- `src/app/api/closing-periods/route.ts`
- `scripts/verify-db-workflow.ts`

## 구현 내용
- `POST /api/closing-periods`에서 기존 마감 월이 있으면 HTTP 409와 `PERIOD_ALREADY_CLOSED`를 반환한다.
- 마감 스냅샷을 바꾸려면 먼저 `DELETE /api/closing-periods`로 마감 해제 후 다시 잠금 처리해야 한다.
- 마감 생성과 감사 로그 기록을 하나의 Prisma 트랜잭션으로 묶었다.
- 마감 해제와 감사 로그 기록도 하나의 Prisma 트랜잭션으로 묶었다.
- DB 워크플로우 검증에 중복 마감 요청이 `PERIOD_ALREADY_CLOSED`로 실패하는 시나리오를 추가했다.

## 실행 명령
- `npm run typecheck`

- `git diff --check`
- `npm run verify:mvp`
- `if [ -n "$DATABASE_URL" ]; then printf 'DATABASE_URL=set\n'; else printf 'DATABASE_URL=missing\n'; fi`
