# 개발 진행 로그 276

## 시점

- 2026-06-21

## 진행 내용

- 접근코드 없이 가능한 남은 갭을 다시 점검했다.
- 운영 DB mutation workflow 자체는 여전히 접근코드와 운영 실행 허용이 필요하므로, DB를 변경하지 않는 사전 검증 단계를 추가했다.
- `verify:railway-authenticated` 스크립트를 추가했다.
  - `RAILWAY_ACCESS_CODE` 또는 `VERIFY_DB_WORKFLOW_ACCESS_CODE`로 `/api/auth/login`에 로그인한다.
  - 로그인 요청에는 대상 URL의 `Origin` 헤더를 붙인다.
  - 인증 쿠키로 `/api/auth/session`, `/api/companies`, `/api/transactions`, `/api/operations/readiness`를 읽는다.
  - 거래/증빙/분개/리포트/마감 데이터는 생성·수정·삭제하지 않는다.
- README와 Railway cutover 문서에 운영 접근코드가 준비된 뒤 실행할 비파괴 인증 읽기 검증 순서를 추가했다.
- 배포 설정 검증에 새 스크립트와 문서 회귀 방지 assertion을 추가했다.

## 검증

- `node --check scripts/verify-railway-authenticated.mjs` 통과.
- `npm run verify:deployment-config` 통과.
- `npm run typecheck` 통과.
- `npm run lint` 통과.
- `npm run verify:mvp` 통과.

## 남은 조건

- 운영 접근코드가 준비되면 먼저 아래 비파괴 검증을 실행한다.

```bash
RAILWAY_ACCESS_CODE=운영_접근코드 npm run verify:railway-authenticated
```

- 그 다음 운영 DB mutation workflow 실행을 명시 허용할 때만 아래 검증을 실행한다.

```bash
VERIFY_DB_WORKFLOW_ALLOW_PRODUCTION=1 VERIFY_DB_WORKFLOW_ACCESS_CODE=운영_접근코드 VERIFY_DB_WORKFLOW_BASE_URL=https://honzang-production.up.railway.app npm run verify:db-workflow
```
