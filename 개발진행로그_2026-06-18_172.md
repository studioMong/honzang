# 개발 진행 로그 172

## 작업 단위
- 남아 있던 보조 API의 직접 `request.json()` 사용을 공통 JSON 파서로 정리했다.
- 코드 커밋(`6a55dfbf2388461cb9faea56d5e9aff65b19268b`) 이후 공개 Railway 배포 상태를 다시 감사했다.

## 구현 내용
- `src/app/api/reviews/route.ts`의 검토 상태 변경 요청에 `parseJsonRequest`를 적용했다.
- `src/app/api/csv-templates/route.ts`의 CSV 매핑 템플릿 삭제 요청에 `parseJsonRequest`를 적용했다.
- `src/app/api/companies/route.ts`의 회사 설정 저장 요청에 `parseJsonRequest`를 적용했다.
- `src/app/api/classification-rules/route.ts`의 자동 분류 규칙 추가/수정/삭제 요청에 `parseJsonRequest`를 적용했다.
- `src/app/api/vendors/route.ts`의 거래처 추가/수정/삭제 요청에 `parseJsonRequest`를 적용했다.
- `src/app/api/imports/route.ts`의 CSV 가져오기/삭제 요청에 `parseJsonRequest`를 적용했다.
- CSV 가져오기 요청은 원본 CSV와 행 JSON을 함께 받으므로 `MAX_ORIGINAL_FILE_TEXT_SIZE + 3_000_000` 바이트까지 허용했다.
- `src/app/api/backups/restore/route.ts`의 백업 복원 요청에 `parseJsonRequest`를 적용했다.
- 백업 복원 요청은 MVP 기준 25MB까지 허용하도록 `MAX_BACKUP_RESTORE_REQUEST_BYTES`를 추가했다.
- `scripts/smoke-production.mjs`에 가져오기, 회사, 검토, 거래처, 자동 분류 규칙, CSV 템플릿 API의 malformed JSON 거부 검증을 추가했다.
- `scripts/verify-backup-restore.mjs`에 백업 복원 API의 malformed JSON 거부 검증을 추가했다.
