## 목표
- 분개에 추가한 `Decimal(14,2)` 금액 검증 기준을 거래, 증빙, CSV 가져오기, 백업 복원 경로까지 확장한다.
- 수기 입력, CSV 입력, 백업 JSON 복원 어디서 들어오더라도 DB 저장 가능 금액 범위와 소수 자릿수 불변식이 동일하게 적용되도록 한다.

## 변경 내용
- `src/lib/server/money-validation.ts`
  - signed 금액 검증 `validateDecimal14_2SignedAmount` 추가
  - 잔액처럼 음수가 가능한 Decimal 필드는 절댓값 한도와 소수 둘째 자리 기준만 확인
- `src/lib/server/transaction-validation.ts`
  - 입금, 출금, 공급가액, 부가세에 `Decimal(14,2)` 비음수 검증 적용
  - 잔액은 signed Decimal 기준으로 검증
  - 공급가액/부가세 합계 비교를 원 단위 반올림이 아니라 1/100 단위로 변경
- `src/lib/server/evidence-validation.ts`
  - 증빙 공급가액, 부가세, 합계에 같은 금액 검증 적용
  - 증빙 합계 일치 비교를 1/100 단위로 변경
- `src/app/api/imports/route.ts`
  - CSV 행의 금액, 입금, 출금, 공급가액, 부가세, 잔액 검증 추가
  - 금액 정밀도 오류가 있으면 저장 전 `INVALID_CSV_ROWS`로 차단
- `scripts/smoke-production.mjs`
  - CSV 가져오기 금액 소수 자릿수 초과 검증 추가
  - 수기 거래 금액 소수 자릿수 초과 검증 추가
  - 증빙 금액 소수 자릿수 초과 검증 추가
- `scripts/verify-backup-restore.mjs`
  - 백업 거래 금액 소수 자릿수 초과 검증 추가
  - 백업 증빙 금액 소수 자릿수 초과 검증 추가
