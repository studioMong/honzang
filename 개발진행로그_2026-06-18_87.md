# 개발 진행 로그 87

## 작업 단위
- 증빙 금액 불일치가 리포트 저장/마감 스냅샷에 보존되는지 확인하는 DB workflow 검증을 강화했다.

## 변경 내용
- `scripts/verify-db-workflow.ts`
  - 실제 API로 증빙 금액 불일치 케이스를 만든 뒤 연결 거래의 `evidenceStatus`가 `ATTACHED`로 바뀌는지 검증한다.
  - `/api/reviews` 호출로 증빙 금액 불일치 리뷰가 생성되는지 검증한다.
  - 생성된 리뷰를 리포트 스냅샷용 CSV 행으로 변환해 `/api/reports` 저장 payload에 포함한다.
  - 저장된 리포트의 `calculatedPayload.reviewItems`에 증빙 금액 불일치 리뷰가 유지되는지 검증한다.
  - 마감 잠금 `summaryPayload.report.reviewItems`에도 같은 리뷰가 유지되는지 검증한다.
  - 연결 증빙 삭제 후 입금 거래의 증빙상태가 `UNCHECKED`로 되돌아오는지 검증한다.
- `scripts/smoke-production.mjs`
  - 리포트 화면 스모크 검증에 `증빙 불일치`, `증빙 금액` 문구를 추가했다.

## 검증

- `npm run verify:mvp` 통과

## 미실행 항목
- `npm run verify:db-workflow`
  - 현재 로컬 셸에 `DATABASE_URL`이 설정되어 있지 않아 실제 DB mutation workflow는 실행하지 않았다.
  - 스크립트는 타입체크와 린트를 통과했으며, DB 모드 앱 또는 staging DB에서 실행해야 한다.
