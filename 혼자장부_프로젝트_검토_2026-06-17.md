## 1. 결론

`혼자장부`는 세무기장 대행 서비스가 아니라, 1인법인 대표가 직접 장부를 정리하고 신고 전 숫자를 검토할 수 있게 돕는 회계/장부 관리 소프트웨어로 잡는 방향이 맞다.

초기 제품의 핵심은 다음 문장이다.

> 대표가 매달 통장, 카드, 홈택스, PG 자료를 올리면 혼자장부가 거래를 분류하고 증빙 누락을 잡아주며, 부가세·원천세·법인세 신고 준비 숫자를 보여준다. 최종 판단과 홈택스 신고는 사용자가 직접 한다.

## 2. 제품 경계

가능한 포지셔닝:
- 1인법인을 위한 셀프 장부 정리 도구
- 거래내역 자동 분류 및 자동 분개 초안
- 증빙 누락 알림
- 부가세, 원천세, 법인세 신고 준비용 집계
- 사용자가 직접 홈택스에 신고하기 위한 참고자료 제공

피해야 할 포지셔닝:
- 세무기장 대행
- 법인세 신고 대행
- 절세 자문
- 세무사가 필요 없는 완전 자동 신고
- 고객 대신 홈택스 신고 제출
- 고객별 세무 판단을 확정해주는 상담/자문형 기능

제품 문구는 `신고 대행`, `절세`, `완전 자동 신고`보다 `장부 정리`, `거래 분류`, `검토 필요`, `신고 준비`, `대표자 직접 신고 지원`에 맞춘다.

## 3. 일반 채팅에서 확인한 핵심

최초 질문은 “스튜디오몽 1인법인용 셀프 세무기장을 만들 수 있는가”였다. 답변의 결론은 내부용 셀프 기장 도구는 충분히 만들 만하지만, 남의 회사 세무기장·신고대리까지 해주는 서비스로 가면 세무대리 영역에 걸릴 수 있다는 것이었다.

그 다음 논의에서 “서비스 자체를 판매하는 경우”도 검토했다. 소프트웨어 판매 자체가 문제의 핵심은 아니고, 서비스가 실제로 세무대리처럼 작동하는지가 핵심이다. 따라서 사용자가 직접 확인하고 신고하는 회계/장부 관리 소프트웨어로 설계해야 한다.

앱 이름은 `혼자장부`가 1순위로 정리됐다. 이유는 1인법인 대표가 듣자마자 “내가 직접 장부 정리하는 앱”이라고 이해할 수 있고, 세무대리 느낌이 상대적으로 적기 때문이다. 최종 사용 전 KIPRIS 상표 검색은 필요하다.

## 4. 실무 흐름

1인법인 실무 흐름은 대략 다음 순서다.

1. 법인통장 거래내역을 내려받는다.
2. 법인카드 사용내역을 내려받는다.
3. 홈택스에서 전자세금계산서, 계산서, 현금영수증, 신용카드 매입자료를 확인한다.
4. 매출이 있으면 세금계산서 발행 여부, 카드/PG 정산 여부, 입금 여부를 맞춘다.
5. 급여, 외주비, 기타소득 지급이 있으면 원천세 대상인지 확인한다.
6. 거래마다 계정과목을 붙인다.
7. 증빙 없는 지출, 개인 사용 의심 지출, 대표자 입출금 같은 위험 거래를 검토한다.
8. 월별 손익, 부가세 예상액, 원천세 납부액을 확인한다.
9. 분기, 반기, 연말에 신고용 자료를 만든다.

혼자장부는 이 흐름을 앱 안에서 순서대로 처리하게 만든다. 사용자는 매달 자료를 올리고, 앱은 자동 분류와 검토 항목을 보여주는 구조다.

## 5. 세금 일정 기준

공식 자료 기준으로 제품 내 일정 관리에 반영할 항목:

- 부가가치세: 일반과세자 기준 1기 예정 4.1~4.25, 1기 확정 7.1~7.25, 2기 예정 10.1~10.25, 2기 확정 다음 해 1.1~1.25.
- 원천세: 일반적으로 소득 지급일이 속하는 달의 다음 달 10일까지. 반기납부는 1~6월분 7.10까지, 7~12월분 다음 해 1.10까지.
- 지급명세서: 근로·퇴직·사업소득 등은 다음 연도 3월 10일. 간이지급명세서 사업소득/기타소득은 지급일이 속하는 달의 다음 달 말일.
- 법인세: 사업연도 종료일이 속하는 달의 말일부터 3개월 이내. 12월 결산법인은 보통 다음 해 3월 31일까지.

세법과 신고기한은 바뀔 수 있으므로 출시 전에는 최신 국세청 안내와 세무사/법률 전문가 검토가 필요하다.

## 6. MVP 범위

첫 버전의 목표는 자동 신고가 아니라 `신고 준비 패키지 생성`이다.

필수 기능:
- 회사 설정: 법인명, 사업자번호, 업종, 과세유형, 결산월, 대표자 급여 여부
- 자료 업로드: 통장 CSV, 카드 CSV, 홈택스 매출/매입자료 CSV, PG 정산 CSV
- 거래 자동 분류: 매출, 광고비, 지급수수료, 소모품비, 통신비, 복리후생비, 외주비, 급여, 세금과공과 등
- 증빙 매칭: 통장/카드 거래와 세금계산서, 현금영수증, 카드전표 연결
- 자동 분개 초안: 차변/대변 분개 생성
- 검토함: 애매한 거래만 사용자가 확인
- 부가세 리포트: 매출세액, 매입세액, 불공제 후보, 납부/환급 예상액
- 원천세 리포트: 급여, 사업소득, 기타소득 지급액과 원천징수세액
- 법인세 준비 리포트: 손익계산서, 재무상태표 초안, 계정별 원장, 증빙 누락 목록
- 마감 체크리스트: 이번 달, 이번 분기, 연말 결산 전 확인할 일

MVP 제외:
- 홈택스 자동 제출
- 인증서/스크래핑 기반 자동수집
- 세무대리 기능
- 절세 자문
- 법인세 신고서 자동 확정

## 7. 개발 순서

1단계: CSV 기반 로컬/웹 MVP

1. 회사 설정
2. 통장 CSV 업로드
3. 카드 CSV 업로드
4. 거래 목록 화면
5. 계정과목 수동 지정
6. 자동 분류 규칙 저장
7. 증빙 파일 첨부
8. 월별 손익 리포트
9. 부가세 집계
10. 원천세 체크리스트
11. 법인세 결산 준비 리포트

2단계: 자동화 강화

1. 은행/카드 CSV 포맷 템플릿 확장
2. 홈택스 내려받기 자료 자동 파서
3. PG사 정산자료 파서
4. 반복 거래 자동 규칙 추천
5. 증빙 자동 매칭 고도화
6. 부가세 신고서 입력값 형태로 출력

3단계: 외부 서비스화

1. 다중 회사/다중 사용자 구조
2. 권한 관리
3. 파일 암호화 및 보안 감사 로그
4. 상표 확인 및 도메인 확보
5. 세무사 제휴 검토
6. 유료 플랜 설계

## 8. 화면 구조

- 대시보드: 오늘 할 일, 미분류 거래 수, 증빙 없는 지출 수, 부가세 예상 납부액, 원천세 신고 필요 여부, 법인세 결산 준비율
- 거래내역: 통장, 카드, PG 거래 전체 관리
- 증빙함: 세금계산서, 계산서, 현금영수증, 카드전표 관리
- 자동분개: 앱이 제안한 분개와 승인/수정/보류 상태 관리
- 신고준비: 부가세, 원천세, 법인세 탭과 다운로드/출력
- 리포트: 손익, 재무상태, 현금흐름, 계정별 원장
- 설정: 계정과목, 자동분류 규칙, 사업자 정보, 결산월, 세금 일정

## 9. 데이터 모델 초안

원본 데이터는 덮어쓰지 않는다. 원본 CSV와 홈택스 자료는 보존하고, 그 위에 분류/분개/검토 상태를 붙인다.

필요 테이블:
- `companies`
- `accounts`
- `transactions`
- `evidences`
- `journal_entries`
- `journal_lines`
- `vendors`
- `tax_reports`
- `review_items`
- `rules`

핵심 원칙:
- 원본 파일과 가공 결과 분리
- 거래 출처 추적 가능성 유지
- 사용자가 확정한 분류와 앱 추천 분류 분리
- 검토 필요 항목을 별도 엔티티로 관리

## 10. 반드시 경고해야 할 항목

- 증빙 없는 비용
- 법인카드 개인 사용 의심 거래
- 대표자에게 나간 돈
- 대표자가 넣은 돈
- 접대비, 경조사비, 식대성 지출
- 해외 SaaS 결제
- 외주비인데 원천세 신고가 없는 건
- 세금계산서 매출은 있는데 입금이 안 된 건
- 입금은 있는데 매출 증빙이 없는 건
- 부가세 공제 불가 가능성이 있는 건
- 3월 법인세 신고 전 재무제표가 맞지 않는 건
- 외부조정 대상 가능성이 있는 건

## 11. 기술/보안 원칙

- 민감정보: 사업자등록번호, 계좌, 거래처, 급여 정보는 민감정보에 준해 다룬다.
- 저장 원칙: 원본 CSV와 업로드 파일은 변경하지 않고 별도 보관한다.
- 감사 가능성: 분류 변경, 분개 승인, 신고 준비 리포트 생성 이력을 남긴다.
- 서비스화 전 필수: 파일 암호화, 접근 로그, 백업/삭제 정책, 권한 관리.
- 초기 배포: 내부용 또는 단일 회사용으로 먼저 검증한다.
- 연 1회 검토: 법인세 결산과 첫 1년 신고는 세무사 1회 검토를 받는 구조가 현실적이다.

## 12. 다음 작업

추천 순서:

1. 샘플 통장 CSV와 카드 CSV 포맷 정의
2. `transactions` 원천 스키마 확정
3. CSV 업로드/파싱/원본 보관 구현
4. 거래 목록 UI 구현
5. 계정과목 수동 지정 구현
6. 자동 분류 규칙 저장 구현
7. 증빙 상태 표시 구현
