# 개발진행로그 82

## 주제

운영 준비 점검 커밋 후 Railway 공개 URL 재진단

## Git 상태

- 커밋: `e4e527171dec6ac9e2a456dfabb375c4307b7083`
- 메시지: `Add operations readiness checks`
- 원격: `origin/main` 푸시 완료

## Railway 확인 결과

```bash
RAILWAY_EXPECTED_COMMIT=e4e527171dec6ac9e2a456dfabb375c4307b7083 npm run verify:railway
```

결과:

- 실패
- `/`: HTTP 200, legacy 정적 프로젝트 요약 페이지
- `/api/version`: HTTP 404
- `/api/health`: HTTP 404
- `/manifest.webmanifest`: HTTP 404

```bash
RAILWAY_AUDIT_SOFT=1 RAILWAY_EXPECTED_COMMIT=e4e527171dec6ac9e2a456dfabb375c4307b7083 npm run audit:railway
```

결과:

- `4 fail, 0 warn, 0 pass`
- Next.js API 라우트 미노출
- legacy 정적 페이지 응답
- healthcheck 미노출
- PWA manifest 미노출

## 판단

운영 준비 점검 API와 설정 화면은 로컬 프로덕션 검증에서 통과했지만, 공개 URL은 여전히 최신 Next.js 서버를 바라보지 않는다. public domain이 이전 static/legacy 서비스에 붙어 있는 상태로 보이며 Railway Dashboard에서 도메인 연결 대상과 최신 Deploy source commit 확인이 필요하다.
