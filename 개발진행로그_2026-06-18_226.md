# 개발 진행 로그 226

## 작업 시각
- 2026-06-18

## 목표
- CSV 금액 정규화에서 입금/출금 금액과 통장 잔액의 부호 처리 기준을 분리한다.
- 입금/출금은 기존처럼 음수 표기를 절댓값 거래금액으로 해석하되, 잔액은 마이너스 통장/대출성 잔액처럼 음수 부호를 보존한다.

## 발견한 문제
- `parseMoney`는 `(1,234)`, `1,234-`, `-1,234` 같은 음수 표기를 모두 절댓값으로 바꾼다.
- 이 처리는 거래 금액에는 적합하지만, CSV 잔액 컬럼에도 같은 함수가 쓰이고 있어 음수 잔액이 양수로 저장될 수 있었다.
- 통장 잔액 대조는 시작/종료 잔액과 거래 순증감을 비교하므로, 잔액 부호가 바뀌면 현금흐름 검증 결과가 왜곡된다.

## 변경 내용
- `src/lib/accounting.ts`
  - `parseSignedMoney` 추가
  - `parseMoney`는 `parseSignedMoney`의 절댓값을 반환하도록 정리
  - `normalizeCsvRow`의 `balance` 정규화는 `parseSignedMoney` 사용
- `src/app/api/imports/route.ts`
  - CSV 행 사전 검증에서 잔액 컬럼은 `parseSignedMoney` 사용
  - 금액/입금/출금/공급가액/부가세는 기존처럼 절댓값 파서 유지
- `scripts/verify-sample-csvs.ts`
  - signed 금액 파서가 괄호형/뒤쪽 마이너스/앞쪽 마이너스/소수 음수를 보존하는지 검증
  - 은행 CSV 정규화에서 출금 금액은 절댓값으로, 잔액은 음수로 보존되는지 검증
