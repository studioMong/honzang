# 개발진행로그 52 - 과금 정책 MVP

## 진행 내용

- `Company`에 과금 단가 필드를 추가했다.
  - `perUseUnitPrice`
  - `monthlySubscriptionPrice`
  - `annualSubscriptionPrice`
- 새 Prisma 마이그레이션 `20260617050000_billing_prices`를 추가했다.
- 회사 설정 화면에서 회당 단가, 월 구독가, 연 구독가를 저장할 수 있게 했다.
- 설정 화면에 `과금 정책` 패널을 추가했다.
  - 선택된 과금 모델
  - 기준 단가
  - 매출 거래 수
  - 매출 공급가액
  - 추정 과금 단위
- 백업 복원 스키마에 새 과금 단가 필드를 포함했고, 기존 백업은 기본값 0으로 복원되도록 했다.
- README MVP 범위에 회당 정산/SaaS 구독 단가 설정과 과금 단위 추정을 반영했다.

## 검증

- `npm run db:generate` 통과
- `npm run typecheck` 통과
- `npm run lint` 통과
- `npm run build` 통과
- `npm run smoke:prod` 통과
- `npm run verify:backup-restore` 통과

## 참고

- 과금 단위 추정은 매출 계정 또는 홈택스 매출성 입금 거래의 공급가액을 기준으로 계산한다.
- 실제 결제/청구 자동화가 아니라 장부 정리와 매출 모델 검토를 위한 MVP 기준값이다.
