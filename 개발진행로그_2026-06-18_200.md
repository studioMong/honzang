# 개발 진행 로그 200

## 작업 주제
- 전체 워크스페이스 백업 JSON/ZIP export 로직을 독립 검증 가능한 라이브러리로 분리.

## 변경 사항
- `src/lib/workspace-backup-export.ts`를 추가했다.
  - 백업 payload 생성, 데이터 보관 정책 행, 백업 준비도 행, 백업 ZIP 파일 목록, export 완료 메시지를 한 곳에서 생성한다.
  - 원본 CSV는 `imports/original-csv/` 경로로, DB 보관 증빙 파일은 `evidences/` 경로로 ZIP에 포함한다.
  - ZIP manifest에 구조 파일, 원본 CSV 포함 현황, DB 증빙 파일 포함 현황, 외부 링크/파일 없는 증빙 수를 기록한다.
- `src/components/app-workspace.tsx`의 백업 다운로드 UI가 새 라이브러리 함수를 사용하도록 정리했다.
  - 앱 화면의 JSON/ZIP 다운로드와 검증 스크립트가 같은 백업 생성 계약을 공유한다.
  - 신고 패키지 ZIP에서 쓰는 증빙 파일 entry 생성도 같은 helper를 재사용한다.
- `scripts/verify-workspace-backup-exports.ts`를 추가했다.
  - 샘플 회사, 원본 CSV, DB 증빙 파일, 마감 기간, 분개, 리포트, 감사 로그 fixture로 백업 payload와 ZIP 파일을 검증한다.
  - manifest, `workspace-backup.json`, CSV BOM, 원본 CSV 파일명 sanitize, 증빙 data URL decode, ZIP byte 포함 여부를 확인한다.
- `package.json`에 `verify:workspace-backup-exports`를 추가했다.
- `verify:mvp`에 `verify:workspace-backup-exports` 단계를 추가했다.

## 검증
- `npm run typecheck` 통과.
- `npm run verify:workspace-backup-exports` 통과.
  - 샌드박스에서는 `tsx` IPC pipe 제한으로 실패했고, 권한 상승 후 동일 명령은 통과했다.
- `npm run lint` 통과.
- `git diff --check` 통과.
- `npm run verify:mvp` 통과.
  - 샌드박스에서는 `verify:data-sources`의 `tsx` IPC pipe 제한으로 중단되었고, 권한 상승 후 전체 단계가 통과했다.

## DB 검증
- 현재 셸에 `DATABASE_URL`이 없어 `npm run verify:db-workflow`는 실행하지 않았다.

## 비고
- 이번 변경으로 전체 백업 다운로드는 UI 수동 확인에만 의존하지 않고, 원본 CSV와 DB 증빙 파일 포함 구조를 자동 검증한다.
- Railway 공개 도메인 legacy/static 연결 문제는 이번 코드 변경과 별개로 계속 남아 있다.
