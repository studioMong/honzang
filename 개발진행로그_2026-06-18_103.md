# 개발 진행 로그 103

## 작업 단위
- 승인된 분개가 있는 거래가 비승인 분개로 덮여 원장 기준이 사라지는 것을 서버에서 차단했다.

## 변경 내용
- `src/app/api/journals/route.ts`
  - 거래에 이미 `APPROVED` 분개가 있고 새 저장 요청이 `DRAFT` 또는 `VOID` 상태이면 409로 거절한다.
  - 응답 코드 `APPROVED_JOURNAL_REPLACEMENT_BLOCKED`와 기존 승인 분개 ID를 반환한다.
  - 승인 분개를 취소하려면 기존 `PATCH /api/journals` 상태 변경 흐름을 사용해야 한다.
- `scripts/verify-db-workflow.ts`
  - 승인 분개 생성 후 같은 거래에 `DRAFT` 분개 저장을 시도하면 409와 `APPROVED_JOURNAL_REPLACEMENT_BLOCKED`가 반환되는지 검증한다.

## 실행 명령
- `npm run typecheck`
- `npm run lint`
- `git diff --check`
- `npm run verify:mvp`

## 검증 결과
- TypeScript typecheck 통과.
- ESLint 통과.
- Git whitespace 검사 통과.

- 현재 셸에 `DATABASE_URL`이 없어 실제 Postgres 대상 `verify:db-workflow`는 실행하지 않았다.
