# 개발진행로그 2026-06-18 236

## 목표

- 잘못 입력한 수기 거래를 거래내역 화면에서 되돌릴 수 있게 한다.
- 수기 공급가액/부가세 거래가 DB 워크플로 검증 계약에도 포함되게 한다.

## 변경

- `src/app/api/transactions/route.ts`
  - `DELETE /api/transactions`를 추가했다.
  - 삭제 대상은 `MANUAL` 거래로 제한했다.
  - CSV 가져오기 거래는 직접 삭제하지 못하게 하고, 업로드 화면의 배치 삭제를 안내한다.
  - 마감된 기간의 거래 삭제는 `PERIOD_CLOSED`로 차단한다.
  - 승인된 분개가 있는 수기 거래 삭제는 차단한다.
  - 삭제 시 연결 증빙은 거래 연결을 해제하고, 연결 검토 항목과 DRAFT/VOID 분개는 정리한다.
  - 삭제 이력은 `TRANSACTION_DELETE` 감사 로그로 남긴다.
- `src/components/app-workspace.tsx`
  - 거래내역 표에 `작업` 컬럼을 추가했다.
  - 수기 거래에만 삭제 버튼을 노출한다.
  - 삭제 성공 시 화면 목록을 갱신하고 DB 모드에서는 서버 상태를 다시 읽는다.
- `scripts/verify-db-workflow.ts`
  - 수기 과세 매출 거래 생성, 공급가액/부가세 저장, 자동분개 초안 생성, 수기 거래 삭제 시 DRAFT 분개 정리를 검증하도록 보강했다.
  - CSV 가져오기 거래 직접 삭제가 `NON_MANUAL_TRANSACTION_DELETE_BLOCKED`로 막히는지 확인한다.
  - 잠긴 기간의 수기 거래 삭제가 `PERIOD_CLOSED`로 막히는지 확인한다.
  - 실패 중간에 남을 수 있는 수기 거래를 cleanup에서 삭제하도록 했다.
- `scripts/verify-period-locks.mjs`
  - 거래 생성, 수정, 삭제 모두 마감 잠금 확인을 요구하도록 갱신했다.
- `scripts/smoke-production.mjs`
  - `DELETE /api/transactions` malformed JSON 거부 검증을 추가했다.
- `scripts/verify-access-control.mjs`
  - 인증 없는 `DELETE /api/transactions`가 401로 막히는지 보호 API 목록에 추가했다.
- `src/lib/workspace-backup-export.ts`, `scripts/verify-workspace-backup-exports.ts`
  - 거래내역 보관/삭제 기준을 `수기 거래 개별 삭제`, `CSV 거래 업로드 배치 삭제`로 구분했다.
- `README.md`
  - MVP 범위에 수기 거래 개별 삭제를 추가했다.

  - 대신 DB 워크플로 스크립트 자체에 수기 거래 생성/삭제 계약을 추가해 스테이징 DB에서 바로 확인 가능하게 했다.

## 메모

- 수기 거래는 사용자가 직접 만든 데이터이므로 개별 삭제를 허용한다.
- CSV 거래는 원본 CSV와 배치 해시, 다중 거래 정합성이 있으므로 기존처럼 업로드 배치 단위로 삭제한다.
