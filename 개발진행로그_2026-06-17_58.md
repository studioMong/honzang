# 개발진행로그 58 - Railway 배포 설정 검증 스크립트

## 진행 내용

- `npm run verify:deployment-config` 명령을 추가했다.
- 검증 스크립트는 다음 배포 설정 회귀를 확인한다.
  - package build/start/db deploy 스크립트
  - `railway.json`의 Dockerfile 빌더, pre-deploy, start, healthcheck 설정
  - Dockerfile의 Node 런타임, `npm ci`, `npm run build`, `EXPOSE 3000`, start CMD
  - Next standalone output 설정
  - `.dockerignore`의 `.next`, `node_modules`, env 파일 제외
  - 루트 `index.html` 부재
  - Prisma schema와 migration 존재
- README의 Railway 검증 절차에 `verify:deployment-config`를 추가했다.

## 검증

- `npm run verify:deployment-config` 통과
- `npm run lint` 통과
- `npm run typecheck` 통과

## 참고

- 이 검증은 저장소 안의 배포 설정 회귀를 잡기 위한 것이다.
- 현재 공개 Railway URL이 기존 정적 페이지를 서빙하는 문제는 도메인/서비스 연결 또는 Railway 대시보드 배포 상태 확인이 여전히 필요하다.
- 커밋 푸시 후 `RAILWAY_EXPECTED_COMMIT=f85c7f4 npm run verify:railway`를 실행했지만 공개 URL은 여전히 기존 정적 프로젝트 요약 페이지를 서빙하고 `/api/version`, `/api/health`, `/manifest.webmanifest`는 404로 응답했다.
