# 개발 진행 로그 - 2026-06-18 35

## 목표
- 수기 거래 입력과 거래 수정 API의 서버 측 데이터 무결성을 보강한다.
- 증빙 상태만 수정할 때 확정 계정과목이 지워질 수 있는 PATCH 회귀를 막는다.

## 변경 내용
- `src/app/api/transactions/route.ts`
  - 수기 거래 생성 시 거래일을 엄격 검증한다.
  - `2026-02-31`처럼 실제로 존재하지 않는 날짜는 `400 INVALID_TRANSACTION_DATE`로 거부한다.
  - PATCH 요청에 `patchTransactionSchema`를 추가해 `id`, `confirmedAccountId`, `evidenceStatus`, `memo`만 허용한다.
  - PATCH에서 전달된 필드만 업데이트하도록 바꿨다.
  - `evidenceStatus`만 수정하는 요청이 기존 `confirmedAccountId`를 `null`로 지우던 문제를 차단했다.
  - PATCH 계정과목 변경 시 계정 존재 여부를 확인한다.
- `scripts/smoke-production.mjs`
  - 잘못된 수기 거래 날짜와 잘못된 PATCH 증빙 상태를 400으로 차단하는지 확인한다.
- `scripts/verify-db-workflow.ts`
  - 실제 DB 모드에서 계정과목 확정 후 증빙 상태만 수정해도 확정 계정이 유지되는지 검증한다.

## 검증
- `npm run typecheck` 통과.
- `npm run lint` 통과.
- `git diff --check` 통과.

## 판단
- CSV 업로드뿐 아니라 수기 거래 입력 경로도 잘못된 날짜 저장을 서버에서 차단한다.
- 거래 수정 API가 부분 업데이트 의미를 지키게 되어, 증빙 상태 변경이 분류 확정 상태를 훼손하지 않는다.
