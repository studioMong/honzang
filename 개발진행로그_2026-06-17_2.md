## Railway 현재 상태

`https://honzang-production.up.railway.app`는 HTTP 200으로 응답하지만, 현재 새 Next.js 앱이 아니라 기존 정적 `index.html` 문서를 서빙하고 있다.

응답 특징:

- `server: railway-hikari`
- `content-length: 24489`
- `/api/health`가 새 앱 응답으로 확인되지 않음

해석:

- GitHub에는 최신 앱 코드가 푸시됐다.
- Railway 서비스가 아직 최신 커밋을 배포하지 않았거나,
- Railway 서비스의 빌더/배포 설정이 정적 HTML 배포 상태로 남아 있을 가능성이 있다.

## Railway에서 확인할 항목

1. Railway 프로젝트가 GitHub 저장소 `studioMong/honzang`의 `main` 브랜치에 연결되어 있는지
2. Root Directory가 저장소 루트인지
3. Build Command가 `npm run build`인지
4. Start Command가 `npm run start`인지
5. Pre-deploy Command가 `npm run db:deploy`인지
6. Variables에 Postgres `DATABASE_URL`이 연결되어 있는지
7. 배포 로그에서 Next.js 빌드가 실행되는지
