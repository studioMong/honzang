# 개발 진행 로그 196

## 작업 주제
- 운영 준비도 API 진단 계약의 독립 검증 추가.

## 변경 사항
- `scripts/verify-operations-readiness.mjs`를 추가했다.
- standalone production 서버를 검증용 환경변수로 실행해 다음 계약을 확인한다.
  - `/api/version`이 production 런타임과 Railway 커밋/브랜치/서비스/환경 메타데이터를 노출한다.
  - `/api/health`가 `DATABASE_URL` 미설정 시 `sample` / `not_configured` 상태를 유지한다.
  - `/api/operations/readiness`가 DB 미설정과 접근코드 미설정을 blocker로 계산한다.
  - 접근 token salt, 공개 앱 URL, production runtime, Railway metadata는 green check로 계산된다.
  - 준비도 check 순서와 label/status/tone/detail/action 텍스트 계약을 고정한다.
- `package.json`에 `verify:operations-readiness`를 추가했다.
- `verify:mvp`에서 production build 직후 `verify:operations-readiness`를 실행하도록 연결했다.

## 검증
- `npm run verify:operations-readiness` 통과.
- `npm run typecheck` 통과.
- `npm run lint` 통과.
- `git diff --check` 통과.
- `npm run verify:mvp` 통과.

## DB 검증
- 현재 셸에 `DATABASE_URL`이 없어 `npm run verify:db-workflow`는 실행하지 않았다.

## 비고
- Railway public domain이 legacy/static 서비스에 연결된 현 blocker 상황에서, 새 검증은 최신 Next.js 서비스가 붙었을 때 `/api/operations/readiness`가 배포 진단용 정보를 안정적으로 제공하는지 보장한다.
