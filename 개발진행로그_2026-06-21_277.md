# 개발 진행 로그 277

## 시점

- 2026-06-21

## 진행 내용

- 사용자의 “개발 완료까지 계속 진행” 요청에 따라 접근코드 없이 가능한 제품 완성도 갭을 추가 점검했다.
- 과금 정책 영역을 보강했다.
  - `sourceType=PG` 거래는 설명에 “정산” 키워드가 없어도 매출 과금 후보로 집계하도록 수정했다.
  - 회당 정산, SaaS 월 구독, SaaS 연 구독 기준의 과금 추정 결과를 재사용 가능한 표 행으로 생성하는 `buildBillingEstimateRows`를 추가했다.
  - 설정 화면의 `과금 정책` 섹션에 과금 추정 상세 표와 `과금 추정표` CSV 다운로드 버튼을 추가했다.
  - smoke 검증에 `과금 추정표`, `PG 매출 후보` 문구를 추가해 설정 화면 회귀를 잡도록 했다.

## 검증

- `npm run verify:billing-estimates` 통과.
- `npm run typecheck` 통과.
- `npm run lint` 통과.
- 로컬 dev 서버 `http://127.0.0.1:3010/?view=settings` HTML에서 `과금 정책`, `과금 추정표`, `PG 매출 후보`, `매출 공급가액` 렌더링 확인.
- Browser 플러그인은 현재 세션에서 `sandboxPolicy` 메타데이터 누락 오류로 제어 도구가 실행되지 않아, 로컬 HTTP 렌더링 검증으로 대체했다.
- `npm run verify:mvp` 통과.
