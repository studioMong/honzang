# 개발진행로그 56 - CSV 가져오기 조건 표시

## 진행 내용

- CSV 업로드 화면의 매핑 영역에 `가져오기 조건` 상태 박스를 추가했다.
- 가져오기 가능 여부를 다음 필수 조건으로 나눠 표시한다.
  - CSV 파일 선택 여부
  - 거래일 컬럼 매핑
  - 내용 컬럼 매핑
  - 금액 컬럼 매핑
- 조건이 충족되지 않으면 `확인 필요` 상태와 각 누락 항목을 보여준다.
- 가져오기 버튼 비활성 기준도 동일한 `importReady` 값으로 정리했다.

## 검증

- `npm run typecheck` 통과
- `npm run lint` 통과
- `npm run build` 통과
- `npm run smoke:prod` 통과
- Browser 데스크톱 검증
  - `/?view=imports`에서 `가져오기 조건` 렌더링 확인
  - 파일 선택 전 `CSV 파일`, `거래일`, `내용`, `금액` 모두 필요 상태 확인
  - 가져오기 버튼 비활성 확인
  - 페이지 전체 가로 오버플로우 없음
  - 콘솔 오류 없음
- Browser 모바일 폭 390px 검증
  - `가져오기 조건` 렌더링 확인
  - 페이지 전체 가로 오버플로우 없음
  - 콘솔 오류 없음

## 참고

- 파일 선택 후 자동 매핑이 성공하면 같은 상태 박스에서 어떤 컬럼이 매핑됐는지 확인할 수 있다.
- 커밋 푸시 후 `RAILWAY_EXPECTED_COMMIT=439203a npm run verify:railway`를 실행했지만 공개 URL은 여전히 기존 정적 프로젝트 요약 페이지를 서빙하고 `/api/version`, `/api/health`, `/manifest.webmanifest`는 404로 응답했다.
