# 개발진행로그 23

## 목표

검토함을 단순 위험 목록이 아니라 처리 상태를 남길 수 있는 업무 흐름으로 확장한다.

## 변경 내용

- 공통 `buildReviewItems` 로직을 `src/lib/accounting.ts`로 이동했다.
- 검토 사유별 심각도와 권장 조치 문구를 공통 함수로 분리했다.
- `/api/reviews`를 추가했다.
- DB 모드에서 현재 거래 기준 검토 후보를 `ReviewItem` 테이블과 동기화한다.
- 더 이상 해당되지 않는 열린 검토 항목은 자동으로 `RESOLVED` 처리한다.
- 검토함 UI에 `처리 완료`, `무시`, `다시 열기` 상태 변경 버튼을 추가했다.
- 대시보드 검토 건수는 열린 항목만 집계하게 했다.
- 거래/증빙 변경 뒤 DB 모드에서는 검토 상태가 다시 계산되도록 refresh를 연결했다.

## 검증 결과

- `npm run typecheck` 통과
- `npm run lint` 통과
- `npm run verify:samples` 통과
- `npm run build` 통과

- 인앱 브라우저에서 검토함 화면, 열린 항목 배지, 상태 버튼, 권장 조치 렌더링 확인
