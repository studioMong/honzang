## 작업 주제
- 신고서 입력값 요약 스냅샷의 DB 저장 검증 보강

## 변경 파일
- `scripts/verify-db-workflow.ts`

## 변경 내용
- DB 워크플로우 검증용 `filingInputSummaryRows` fixture를 추가했다.
- 리포트 저장 payload의 `calculatedPayload.filingInputSummaryRows`에 신고서 입력값 요약 행을 포함했다.
- 월마감 저장 payload의 `summaryPayload.report.filingInputSummaryRows`에도 같은 요약 행을 포함했다.
- 저장 후 `/api/reports`, `/api/closing-periods` 재조회 결과에서 `과세 매출 공급가액` 행과 `집계됨` 상태가 보존되는지 확인하는 assertion을 추가했다.

## 검증 결과
- `npm run typecheck`: 통과
- `npm run lint`: 통과
- `git diff --check`: 통과
- `npm run verify:mvp`: 통과
  - 최초 sandbox 실행은 `tsx` IPC pipe listen 제한으로 실패했다.
  - 승인 실행에서는 전체 MVP 검증이 통과했다.

## 실행하지 못한 검증
- `npm run verify:db-workflow`
  - 현재 쉘 환경에 `DATABASE_URL`이 없어 실제 Railway/Postgres 연결 검증은 실행하지 않았다.
  - DB 연결 문자열이 제공되면 같은 스크립트가 리포트와 마감 payload의 신고서 입력값 요약 보존 여부까지 검증한다.
