# 개발 진행 로그 35

## 목표

- 신고 준비자료를 개별 CSV/JSON으로만 받는 불편을 줄이고, 한 번에 전달 가능한 패키지 파일을 만든다.

## 작업 내용

- 리포트 화면의 신고 준비 영역에 `패키지 ZIP` 다운로드 버튼을 추가했다.
- 신고 패키지 영역에도 `ZIP` 버튼을 추가했다.
- ZIP 안에 다음 파일을 포함하도록 구성했다.
  - `manifest.json`
  - `filing-package.json`
  - `csv/filing-schedule.csv`
  - `csv/filing-package.csv`
  - `csv/transactions.csv`
  - `csv/evidences.csv`
  - `csv/vat-report.csv`
  - `csv/review-items.csv`
  - `csv/withholding-candidates.csv`
  - `csv/corporate-tax-prep.csv`
  - `csv/financial-statements.csv`
  - `csv/ledger.csv`
- 외부 의존성 없이 브라우저에서 ZIP 바이트를 만드는 `src/lib/zip.ts` 유틸을 추가했다.
- ZIP 내부 CSV도 엑셀에서 바로 열기 쉽도록 UTF-8 BOM을 붙인다.
- README의 MVP 범위를 `신고 준비자료 JSON/CSV ZIP 패키지 다운로드`로 갱신했다.

## 검증

- `npm run typecheck` 통과
- `npm run lint` 통과
- `npm run verify:samples` 통과
- `npm run build` 통과
- `npm run smoke:prod` 통과
- ZIP 유틸 검증

  - `unzip -l /tmp/honzang-zip-test.zip`으로 `manifest.json`, `csv/test.csv` 목록 확인
- 로컬 production 서버 리포트 화면 SSR 확인
  - `/?view=reports` 200 응답
  - `패키지 ZIP` 문구 확인
  - `ZIP` 문구 확인

## 비고

- ZIP은 무압축 저장 방식이다. 신고 준비자료는 CSV/JSON 텍스트 중심이라 MVP에서는 충분하고, 의존성 추가 없이 브라우저에서 안정적으로 생성할 수 있다.
- 실제 대용량 증빙 원본까지 ZIP에 포함하는 기능은 오브젝트 스토리지 도입 이후 별도 설계가 필요하다.
