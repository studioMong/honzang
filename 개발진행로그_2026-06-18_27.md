# 개발 진행 로그 - 2026-06-18 27

## 목표
- CSV 가져오기 API가 UI를 우회한 잘못된 매핑 요청을 서버에서 차단하게 한다.
- 거래일/내용/금액 매핑이 누락되거나 CSV 헤더에 없는 컬럼을 참조하는 경우 거래 생성 전에 거부한다.

## 변경 내용
- `src/app/api/imports/route.ts`
  - `validateImportMapping` 서버 검증을 추가했다.
  - 필수 매핑: 거래일, 내용/적요, 금액 또는 입금/출금.
  - `headers`가 전달된 경우 모든 매핑 컬럼이 실제 CSV 헤더에 존재하는지 확인한다.
  - 실패 시 `400 INVALID_CSV_MAPPING`과 사용자 확인용 `issues`를 반환한다.
- `scripts/smoke-production.mjs`
  - 잘못된 CSV 매핑 import 요청이 400으로 차단되는지 프로덕션 스모크에 추가했다.

## 검증
- `npm run typecheck` 통과.
- `npm run lint` 통과.
- 샌드박스 내 `npm run verify:mvp`는 `tsx` IPC 파이프 생성 권한 문제로 실패했다.
- 외부 실행 `npm run verify:mvp` 통과.

## 판단
- CSV 업로드 UI의 `canImport` 검증과 서버 API 계약이 맞춰졌다.
- API 직접 호출로 빈 매핑 또는 존재하지 않는 헤더 매핑이 들어와도 오늘 날짜/기본 설명/0원 거래가 저장되는 경로를 차단했다.
