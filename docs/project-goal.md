# 혼자장부 최종 목표

## 제품 목표

혼자장부는 1인법인 대표가 직접 장부를 정리하고, 홈택스 신고 전에 필요한 대조자료와 신고 준비 패키지를 만들 수 있게 돕는 웹/PWA 서비스다.

최종 사용자는 세무 지식이 깊지 않아도 통장, 카드, 홈택스, PG 자료를 모아 거래를 정리하고, 증빙과 분개를 확인한 뒤, 부가세·원천세·지급명세서·법인세 신고 전에 확인해야 할 숫자와 위험 항목을 순서대로 볼 수 있어야 한다.

## 100% 완료 기준

- Railway 공개 URL에서 최신 GitHub `main` 커밋의 Next.js standalone 앱이 응답한다.
- Railway Postgres가 연결되어 `/api/health`에서 `database=connected`로 확인된다.
- 앱 화면과 민감 API는 접근코드로 보호되고, 익명 사용자는 `/access`로 이동하거나 HTTP 401을 받는다.
- 통장, 카드, 홈택스 매출/매입, 현금영수증, PG CSV 샘플이 파싱되고 실제 컬럼 매핑 흐름을 통과한다.
- 원본 CSV, 증빙 파일, 거래, 거래처, 분류 규칙, CSV 매핑 템플릿, 분개, 검토 항목, 리포트, 마감 스냅샷이 DB 모드에서 저장·조회·수정·삭제 규칙을 통과한다.
- 월 마감 잠금 이후 확정 기간의 거래, 증빙, 분개, 리포트, CSV 삭제가 차단된다.
- 신고 일정, 홈택스 제출 전 입력 가이드, 신고서 입력값 요약, 법인세 외부조정 검토가 국세청 근거 링크와 함께 제공된다.
- 신고 패키지는 브라우저 인쇄/PDF 저장, CSV, JSON, ZIP, XLSX로 내려받을 수 있고, 국세청 근거 링크도 export에 보존된다.
- 전체 워크스페이스 백업 JSON/ZIP을 내려받을 수 있고, 백업 복원은 dry-run과 명시 확인 문구 없이는 실행되지 않는다.
- 운영 준비 점검에서 Postgres, 스키마, 접근코드, 접근 쿠키 salt, 파일 암호화 키, 공개 앱 URL, Railway 메타데이터 상태가 확인된다.
- `npm run verify:mvp`, `npm run verify:railway`, `npm run verify:railway-access`가 통과한다.
- DB mutation workflow는 접근코드 보호 환경에서 `VERIFY_DB_WORKFLOW_ACCESS_CODE`로 인증 후 실행할 수 있으며, 운영 DB에서는 명시적인 `VERIFY_DB_WORKFLOW_ALLOW_PRODUCTION=1` 없이는 실행되지 않는다.

## 제품 경계

혼자장부는 신고 준비 도구다. 다음은 제품 범위에서 제외한다.

- 세무기장 대행
- 세무 상담 또는 절세 자문
- 홈택스 자동 제출
- 신고 대행
- 법인세 신고서 자동 확정
- “세무사 없이 완벽 신고” 같은 단정적 문구

앱은 계산 결과와 검토표를 제공하지만, 최종 신고 여부와 입력값 확정은 사용자가 홈택스, 국세청 공지, 필요 시 세무 전문가 검토를 통해 결정한다.

## 현재 남은 외부 조건

- 운영 DB mutation workflow를 실제 운영 URL에서 실행하려면 `VERIFY_DB_WORKFLOW_ACCESS_CODE` 값이 필요하다.
- 운영 DB workflow는 테스트 데이터를 생성하고 정리하므로, 운영 DB에서 실행할 때는 사용자가 의도적으로 허용해야 한다.
