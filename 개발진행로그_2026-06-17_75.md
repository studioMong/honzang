# 개발진행로그 75

## 작업 시각

- 2026-06-17 23:31 KST

## 목표

- 세무자료와 증빙 파일을 다루는 앱으로서 민감 데이터의 보관 위치와 삭제 경로를 앱 안에서 확인할 수 있게 한다.
- 증빙 파일을 DB에 보관할 수만 있고 개별 삭제할 수 없는 운영 공백을 줄인다.

## 작업 내용

- 증빙 삭제 API `DELETE /api/evidences`를 추가했다.
- 증빙 삭제 시 마감 잠금된 발행일 또는 연결 거래 기간이면 `PERIOD_CLOSED`로 차단한다.
- 증빙 삭제 시 연결 거래에 남은 증빙이 없으면 거래 증빙 상태를 `MISSING` 또는 `UNCHECKED`로 되돌린다.
- 증빙 삭제 활동 로그 `EVIDENCE_DELETE`를 기록한다.
- 증빙 목록에 `삭제` 버튼을 추가했다.
- 설정 화면에 `데이터 보관/삭제 기준` 표를 추가했다.
- 보관/삭제 기준은 다음 항목을 포함한다.
  - 원본 CSV
  - 거래내역
  - 증빙 파일
  - 분개/원장
  - 리포트/마감
  - 거래처/분류 규칙
  - 검토/감사 로그
  - 백업 파일
- 전체 백업 JSON에 `dataRetentionRows`를 포함했다.
- 전체 백업 ZIP에 `csv/data-retention-policy.csv`를 포함했다.
- `verify:backup-restore`가 설정 화면의 데이터 보관/삭제 기준 문구를 확인하도록 확장했다.
- `verify:db-workflow`에 증빙 생성, 마감 중 삭제 차단, 마감 해제 후 삭제 성공, 감사 로그 검증을 추가했다.
- README의 MVP 범위에 증빙 개별 삭제와 데이터 보관/삭제 기준을 추가했다.
