## 구현 내용

- 리포트 화면 상단에 `최종 신고 점검` 패널 추가.
- 점검 순서:
  1. 자료 수집
  2. 거래 분류
  3. 증빙
  4. 부가세
  5. 원천세/대표자
  6. 자동분개/원장
  7. 월 마감
- 각 항목에 상태, 근거, 다음 작업을 표시.
- 차단 항목 수와 확인 항목 수를 상단 배지로 표시.
- 통장 자료나 거래 자체가 없으면 자료 수집을 차단으로 표시.
- 카드, 홈택스, 현금영수증 등 사업 형태에 따라 없을 수 있는 보조 자료는 확인 항목으로 표시.
- 저장된 세무 리포트 상세에도 저장 당시 최종 신고 점검 결과를 표시.
- 신고 패키지 JSON payload와 `tables.filingReadiness`에 최종 점검 결과 포함.
- ZIP 패키지에 `csv/filing-readiness.csv` 추가.
- XLSX 통합 문서에 `최종점검` 시트 추가.
- XLSX 요약 시트에 신고 차단 항목 수와 확인 항목 수 추가.
- README MVP 범위에 최종 신고 점검 항목 추가.
