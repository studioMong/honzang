# 개발진행로그 60 - MVP 통합 검증 명령

## 진행 내용

- `npm run verify:mvp` 명령을 추가했다.
- 이 명령은 로컬/저장소 기준 MVP 회귀 검증을 순서대로 실행한다.
  - Railway/standalone 배포 설정 검증
  - TypeScript 타입체크
  - ESLint
  - CSV 샘플 파싱/분류 검증
  - 프로덕션 standalone 빌드
  - PWA 리소스와 설치 훅 검증
  - 백업 복원 dry-run 검증
  - 프로덕션 API/UI 스모크 테스트
- 실제 DB를 변경하는 `verify:db-workflow`는 통합 검증에서 제외하고, 로컬/스테이징 DB 전용 별도 명령으로 유지했다.
- README에 `verify:mvp`와 DB 저장 흐름 검증의 차이를 반영했다.

## 검증

- `npm run verify:mvp` 통과

## 참고

- `verify:mvp`는 배포 가능한 저장소 상태를 빠르게 확인하기 위한 로컬 게이트다.
- Railway 공개 URL이 최신 Next.js 서비스와 Postgres에 연결됐는지는 `npm run verify:railway`로 별도 확인해야 한다.
