# 개발진행로그 17

## 목표

반복 거래처와 서비스 결제를 사용자가 직접 자동 분류 규칙으로 관리할 수 있게 한다.

## 변경 내용

- `AppClassificationRule` 타입을 추가했다.
- `ClassificationRule` DB 모델을 앱에서 사용할 수 있도록 직렬화 함수를 추가했다.
- `/api/classification-rules` API를 추가했다.
  - `GET`: 규칙 목록 조회
  - `POST`: 키워드 기반 규칙 생성
  - `PATCH`: 활성 상태, 키워드, 계정과목 등 수정
  - `DELETE`: 규칙 삭제
- 회사 초기 조회 응답에 자동 분류 규칙 목록을 포함했다.
- 설정 화면에 자동 분류 규칙 추가 폼과 목록을 추가했다.
- CSV 업로드 미리보기와 서버 import 저장 과정에서 사용자 규칙을 적용하도록 연결했다.

## MVP 규칙 범위

- 조건: 거래 설명 또는 거래처에 키워드 포함
- 선택 범위: 전체 자료 또는 특정 자료 유형
- 액션: 지정 계정과목을 추천 계정으로 적용
- 우선순위: 숫자가 낮을수록 먼저 적용

## 검증 결과
