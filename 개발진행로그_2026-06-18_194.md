# 개발 진행 로그 194

## 작업 주제
- 마감 스냅샷 JSON/ZIP/XLSX export 패키지의 독립 검증 추가.

## 변경 사항
- `src/lib/table-export.ts`를 추가해 CSV 직렬화와 UTF-8 BOM 처리, CSV 수식 주입 방지 규칙을 공용 함수로 분리했다.
- `src/lib/closing-snapshot-export.ts`를 추가해 마감 스냅샷 export payload, ZIP manifest/CSV 파일 목록, XLSX sheet 구성을 UI와 검증 스크립트가 같은 경로로 사용하도록 분리했다.
- `src/components/app-workspace.tsx`의 마감 스냅샷 다운로드 로직을 새 공용 builder 호출로 교체했다.
- `scripts/verify-closing-snapshot-exports.ts`를 추가해 샘플 마감본 기준으로 다음을 직접 검증한다.
  - export payload의 앱명, 유형, 마감 기간, 카운트, 요약 행.
  - ZIP manifest와 파일 순서.
  - CSV BOM, 신고 입력값 라벨, CSV 수식 주입 방지.
  - ZIP 바이트 내부 파일명과 핵심 CSV 내용.
  - XLSX workbook sheet 이름, worksheet 파일, 신고 입력값 라벨.
- `verify:mvp`에 `verify:closing-snapshot-exports` 단계를 추가했다.

## 검증
- `npm run verify:closing-snapshot-exports` 통과.
- `npm run typecheck` 통과.
- `npm run lint` 통과.
- `git diff --check` 통과.
- `npm run verify:mvp` 통과.

## DB 검증
- 현재 셸에 `DATABASE_URL`이 없어 `npm run verify:db-workflow`는 실행하지 않았다.

## 비고
- 이번 변경은 다운로드 UI의 버튼 존재 확인을 넘어, 실제 export 패키지의 파일 구조와 핵심 내용을 CLI에서 재현 가능하게 검증하는 단계다.
- Railway 공개 도메인 반영 여부는 커밋/푸시 후 별도 감사 로그로 기록한다.
