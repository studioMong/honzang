# 개발 진행 로그 97

## 작업 단위
- 프로덕션 접근 쿠키 salt 누락 시 기본 salt로 로그인 쿠키가 발급되지 않도록 접근제어를 강화했다.

## 변경 내용
- `src/lib/server/access-control.ts`
  - `isAccessTokenSaltConfigured()`를 추가했다.
  - 프로덕션에서는 `HONZANG_ACCESS_TOKEN_SALT`가 없으면 접근 토큰을 생성하지 않는다.
  - 개발 환경에서만 기본 salt fallback을 허용한다.
- `src/app/api/auth/login/route.ts`
  - 접근 토큰 생성 실패 메시지를 접근 코드와 쿠키 salt 설정 확인으로 조정했다.
- `src/app/api/operations/readiness/route.ts`
  - 프로덕션에서 salt가 없으면 `접근 쿠키 salt` 점검을 red 차단 항목으로 표시한다.
- `README.md`
  - Railway 필수 환경변수에 `HONZANG_ACCESS_TOKEN_SALT`를 포함했다.
  - 프로덕션에서는 salt가 있어야 접근 쿠키를 발급한다고 명시했다.
- `scripts/verify-deployment-config.mjs`
  - production salt 필수 정책과 운영 준비 점검 문구를 회귀 검사한다.
- `scripts/verify-access-control.mjs`
  - 접근 쿠키의 `HttpOnly`, `Secure`, `SameSite=Lax`, 7일 만료를 검증한다.
  - 운영 준비 점검에서 `accessCode`, `accessSalt`가 green인지 확인한다.

## 실행 명령
