# 개발 진행 로그 - 2026-06-17 5차

## 이번 목표
- Railway가 기존 정적 HTML 대신 Next.js 앱을 빌드/서빙하도록 배포 감지 충돌을 정리한다.

## 확인한 원인
- 배포 URL `https://honzang-production.up.railway.app`의 HTML 본문이 루트 `index.html`과 동일했다.
- 응답 제목은 `혼자장부 프로젝트 정리`였고, `content-length: 24489`도 루트 `index.html` 크기와 일치했다.
- 루트에 정적 `index.html`이 있으면 Railway/Railpack이 정적 사이트로 감지할 수 있으므로 Next 앱 배포에 방해가 된다.

## 조치
- 루트 `index.html`을 `attachments/legacy-project-summary.html`로 이동했다.
- 원본 기획 HTML은 보존하되, 저장소 루트에서는 제거했다.
- README 배포 섹션에 루트 정적 `index.html` 금지 조건을 추가했다.
- 프로젝트 검토 문서의 출처 경로를 새 위치로 수정했다.

## 검증 기준
- 로컬에서는 `npm run build`가 계속 통과해야 한다.
- 원격 배포 후 `https://honzang-production.up.railway.app/api/health`가 JSON으로 응답해야 한다.
- 원격 배포 후 메인 페이지 제목/본문이 기존 `혼자장부 프로젝트 정리` 정적 문서가 아니라 새 앱 UI여야 한다.

## 다음 확인
- 커밋/푸시 후 Railway가 새 커밋을 자동 배포하는지 확인한다.
- 자동 배포가 여전히 정적 HTML이면 Railway 대시보드에서 서비스 연결 브랜치, Root Directory, Build/Start 설정, Variables를 확인해야 한다.
