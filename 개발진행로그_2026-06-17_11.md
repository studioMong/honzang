# 개발진행로그 11

## 목표

샘플 CSV가 실제 업로드 매핑/정규화 로직을 계속 통과하는지 자동 검증한다.

## 변경 내용

- 업로드 화면에만 있던 CSV 컬럼 자동 추론 함수를 `src/lib/accounting.ts`의 공용 함수로 이동했다.
- 은행/카드/홈택스/현금영수증/PG 샘플 CSV를 검증하는 `npm run verify:samples` 명령을 추가했다.
- PG 정산 CSV의 `정산금액`은 매출 입금 성격이므로 출금이 아니라 입금으로 정규화하도록 수정했다.
- `정산`, `구독 결제` 키워드를 매출 계정 추론 후보에 추가했다.

## 검증 기준

- 각 샘플 CSV가 파싱 오류 없이 읽혀야 한다.
- 자료 유형별 필수 컬럼이 자동 추론되어야 한다.
- 행 수, 입금 합계, 출금 합계가 예상값과 일치해야 한다.
- 홈택스 매출과 PG 정산은 매출 공급가액과 부가세예수금 계산값이 일치해야 한다.

## 검증 결과

- 최초 검증에서 PG 샘플의 `정산일` 헤더가 날짜 컬럼으로 추론되지 않는 문제가 발견되어 키워드를 추가했다.
- `npm run verify:samples` 통과
- `npm run lint` 통과
- `npm run typecheck` 통과
- `npm run build` 통과
