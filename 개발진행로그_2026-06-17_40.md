# 개발 진행 로그 40

## 목표

- 원본 CSV를 덮어쓰지 않고 보관한다는 제품 원칙을 DB와 업로드 화면에 반영한다.

## 작업 내용

- `ImportBatch`에 원본 CSV 보관 필드를 추가했다.
  - `originalFileMimeType`
  - `originalFileSize`
  - `originalFileText`
- Railway Postgres 배포용 마이그레이션을 추가했다.
  - `prisma/migrations/20260617020000_import_batch_original_file/migration.sql`
- CSV 업로드 시 2MB 이하 원본 CSV 텍스트를 함께 저장하도록 했다.
- 업로드 이력 목록에는 원문을 싣지 않고 `hasOriginalFile`, 파일 크기 같은 메타 정보만 내려주도록 했다.
- `GET /api/imports?importBatchId=...`로 보관된 원본 CSV를 단건 조회할 수 있게 했다.
- 최근 업로드 표에 `원본` 버튼을 추가해 보관된 CSV를 다시 내려받을 수 있게 했다.
- 중복 업로드가 기존 배치와 매칭되고 기존 배치에 원본이 없으면 새 요청의 원본 CSV로 보강 저장하도록 했다.
- `verify-db-workflow`에 원본 CSV 보관 및 단건 조회 검증을 추가했다.
- README의 MVP 범위에 원본 CSV DB 보관 및 업로드 이력 원본 다운로드를 추가했다.

## 검증

- `npm run db:generate` 통과
- `npm run typecheck` 통과
- `npm run lint` 통과
- `npm run verify:samples` 통과
- `npm run build` 통과
- `npm run smoke:prod` 통과
- `npm run verify:pwa` 통과
- 로컬 production 서버 업로드 화면 SSR 확인
  - `/?view=imports` 200 응답
  - `CSV 업로드` 문구 확인
  - `원본` 문구 확인
  - 원본 보관 제한 크기 문구 확인

## 비고

- 실제 DB 쓰기 흐름의 원본 CSV 보관 검증은 `verify-db-workflow`에 추가했지만, 이 검증은 로컬 또는 스테이징 Postgres 연결 환경에서 실행해야 한다.
- 현재 MVP 제한은 원본 CSV 2MB 이하이다. 대용량 업로드는 추후 파일 스토리지 분리 또는 스트리밍 업로드 설계가 필요하다.
