# 개발진행로그 2026-06-18 242

## 목표

- 증빙을 삭제 후 재등록하지 않고 증빙함 수정 모드에서 DB 보관 파일까지 교체할 수 있게 한다.
- 파일 교체도 기존 암호화 저장, 파일 검증, 월 마감 잠금, 백업 정책과 같은 운영 계약 안에 둔다.

## 변경

- `src/app/api/evidences/route.ts`
  - `PATCH /api/evidences`가 `fileName`, `fileUrl`, `fileDataUrl`, `fileMimeType`, `fileSize`를 받을 수 있게 확장했다.
  - 파일 교체 요청은 기존 `validateEvidenceFile`, `validateEvidenceFileUrl`을 통과해야 한다.
  - 파일 본문은 `encryptStoredText(payload.fileDataUrl)`로 다시 암호화해 `Evidence.rawPayload.fileDataUrl`에 저장한다.
  - 파일 데이터 없이 MIME 또는 크기만 바꾸는 부분 수정은 `INVALID_EVIDENCE_FILE`로 차단한다.
  - 기존 `rawPayload` 객체는 보존하고 파일 관련 키만 갱신한다.
- `src/components/app-workspace.tsx`
  - 증빙 수정 모드에서도 파일 선택을 허용했다.
  - 기존 파일 본문은 기본 PATCH에 다시 싣지 않고, 사용자가 새 파일을 선택하거나 파일 제거를 눌렀을 때만 파일 필드를 전송한다.
  - 수정 모드에 `파일 제거` 버튼을 추가했다.
- `scripts/verify-db-workflow.ts`
  - 증빙 PATCH로 파일을 교체하고, 응답의 `fileDataUrl`이 교체한 data URL과 일치하는지 검증한다.
  - 파일명, MIME, 크기 메타데이터가 함께 갱신되는지 검증한다.
- `scripts/smoke-production.mjs`
  - `PATCH /api/evidences`가 MIME 불일치 파일 payload를 `INVALID_EVIDENCE_FILE`로 거부하는지 검증한다.
- `scripts/verify-file-encryption.ts`
  - 증빙 최초 업로드와 파일 교체가 모두 암호화 저장 경로를 쓰는지 확인하도록 강화했다.
- `README.md`, `src/lib/workspace-backup-export.ts`, `scripts/verify-workspace-backup-exports.ts`
  - 증빙 보관/삭제 기준을 메타데이터/파일 수정 흐름으로 갱신했다.

## 메모

- 이번 변경으로 증빙 수정은 메타데이터, 연결 거래, DB 보관 파일 교체까지 처리한다.
