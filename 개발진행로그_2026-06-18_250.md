# 개발진행로그 2026-06-18 250

## 별도 채팅 확인
- 확인한 스레드: `셀프 세무기장 가능 여부`
- 최초 목적: 스튜디오몽 1인법인용 셀프 장부 정리 도구를 만들 수 있는지 검토
- 결론: 내부용 장부 정리 도구와 사용자 직접 신고 지원 SaaS는 가능성이 있으나, 세무기장 대행/신고대행/절세 자문처럼 보이면 세무대리 영역 리스크가 커진다.

## 제품 경계
- 유지할 방향
  - 1인법인 대표가 직접 장부를 정리하는 도구
  - 거래 CSV 업로드, 자동 분류, 자동 분개 초안, 증빙 누락 알림
  - 부가세/원천세/법인세 신고용 숫자 집계와 검토표 생성
  - 홈택스 신고는 사용자가 직접 진행
  - 애매한 항목은 `추천`, `주의`, `검토 필요`로 표시
- 피할 방향
  - 세무기장 대행
  - 신고 대행
  - 세무 상담/절세 자문
  - 고객 대신 홈택스 제출
  - “세무사 없이 완벽 신고” 같은 단정적 문구

## 현재 구현 반영 상태
- 회사 설정, 과금 모델, CSV 가져오기, 거래 목록, 증빙함, 자동 분개 초안, 검토함, 신고 리포트, 마감/백업/감사 로그까지 MVP 흐름이 구현되어 있다.
- 검토함과 활동 로그는 사용자의 최종 확인 구조를 강화하므로 별도 채팅에서 정한 제품 경계와 맞다.
- Railway 배포만 아직 공개 도메인이 legacy/static 서비스에 연결되어 있어 운영 반영이 막혀 있다.

## 앞으로 유지할 개발 원칙
- 기능명과 화면 문구는 `장부 정리`, `신고 준비`, `검토`, `리포트` 중심으로 유지한다.
- 세무 판단이 필요한 결과는 확정 단정하지 않고, 사용자가 확인할 수 있는 근거와 위험 신호를 보여준다.
- 법인세 자동 제출은 MVP 범위 밖으로 유지하고, 결산 준비 패키지와 전문가 검토 가능성 안내를 우선한다.
