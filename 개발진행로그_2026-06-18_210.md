# 개발 진행 로그 210

## 작업 주제
- 접근 보안 감사 로그의 경계 이벤트를 보강했다.

## 구현 내용
- 이미 잠금 상태인 접속 출처가 다시 로그인 요청을 보내면 `ACCESS_LOGIN_LOCKED` 감사 이벤트를 남기도록 했다.
- 잘못된 로그인 요청 형식도 `ACCESS_LOGIN_FAILURE` 감사 이벤트로 기록하도록 했다.
- 감사 metadata에는 다음 사유만 저장한다.
  - `already_locked`
  - `content_length_too_large`
  - `body_too_large`
  - `invalid_json`
  - `invalid_schema`
- 기존 원칙대로 원문 IP, user-agent, 접근 코드는 저장하지 않는다.
- README의 접근 감사 설명에 잘못된 요청 형식과 잠금 상태 재시도 기록을 반영했다.

## 검증 보강
- `scripts/verify-access-audit.ts`에서 이미 잠긴 출처, 잘못된 JSON, 스키마 오류, 과대 본문 감사 분류 계약을 확인하도록 확장했다.
