# 개발진행로그 77

## 주제

배포 환경 접근코드 보호 추가

## 변경 내용

- `HONZANG_ACCESS_CODE`가 설정된 실행 환경에서 웹 화면과 민감한 `/api/*` 라우트를 접근코드로 보호하도록 `src/proxy.ts`를 추가했다.
- `/access` 로그인 화면과 `/api/auth/login`, `/api/auth/logout`, `/api/auth/session` 라우트를 추가했다.
- `/api/health`, `/api/version`, PWA 리소스, 샘플 CSV는 Railway 점검과 앱 설치를 위해 공개 상태로 유지했다.
- 앱 상단에 보호 모드에서만 보이는 `접근 종료` 버튼을 추가했다.
- `HONZANG_ACCESS_TOKEN_SALT` 권장 환경변수와 접근 쿠키 유지 정책을 README에 문서화했다.
- `npm run verify:access-control`을 추가하고 `npm run verify:mvp`에 포함했다.

## 검증

- `npm run typecheck`: 통과
- `npm run lint`: 통과
- `npm run build`: 통과
- `npm run verify:access-control`: 통과
- `npm run verify:mvp`: 통과
- 브라우저 확인:
  - `/` 접근 시 `/access?next=...`로 리다이렉트 확인
  - 접근코드 입력 후 원래 `/?view=reports` 화면으로 복귀 확인
  - 상단 `접근 종료` 버튼 노출 확인
  - 390px 모바일 폭에서 접근 화면 가로 오버플로 없음 확인

## Railway 설정 메모

Railway Variables에 아래 값을 추가해야 실제 배포 환경에서 보호가 켜진다.

```bash
HONZANG_ACCESS_CODE=배포_접근_코드
HONZANG_ACCESS_TOKEN_SALT=쿠키_서명용_긴_랜덤값
```
