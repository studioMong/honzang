# 개발 진행 로그 178

## 작업 시각
- 2026-06-18

## 작업 주제
- 자료 수집 현황 계산 로직 공통화
- 리포트 화면의 업로드 이력/원본 CSV 보존 상태 표시
- DB 워크플로우 검증에 실제 업로드 배치 기준 단언 추가

## 구현 내용
- `src/lib/data-sources.ts`를 추가해 자료 수집 현황 행 생성 로직을 UI 컴포넌트 밖의 순수 함수로 분리했다.
- `DATA_SOURCE_TYPES`와 `buildDataSourceRows()`를 대시보드와 리포트 화면이 함께 사용하도록 연결했다.
- 리포트 화면과 저장된 리포트 스냅샷의 `자료 수집 현황` 표에 `업로드`, `원본` 열을 추가했다.
- `ReportsPanel`이 `importBatches`를 받도록 변경해 리포트/CSV 내보내기 데이터에도 업로드 이력과 원본 CSV 보존 상태가 포함되게 했다.
- `scripts/verify-db-workflow.ts`에서 실제 `/api/imports` 배치 목록을 조회한 뒤 다음 조건을 검증하도록 보강했다.
  - 통장 자료: 거래 2건, 업로드 2개, 원본 CSV 2/2개
  - 카드 자료: 거래 1건, 업로드 1개, 원본 CSV 1/1개
  - 홈택스 매출 자료: 미업로드 상태와 원본 CSV 없음
  - PG 정산 자료: 선택 자료로 유지
