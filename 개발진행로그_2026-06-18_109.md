# 개발 진행 로그 109

## 작업 단위
- 승인된 분개가 있는 거래에 새 분개를 다시 POST해서 기존 승인 분개를 덮어쓰는 경로를 차단했다.
- 기존에는 같은 거래의 승인 분개를 `DRAFT` 등 비승인 분개로 교체하는 경우만 막았고, 새 `APPROVED` 분개로 재작성하는 경우는 막지 못했다.

## 변경 파일
- `src/app/api/journals/route.ts`
- `scripts/verify-db-workflow.ts`

## 구현 내용
- `POST /api/journals`에서 거래에 이미 `APPROVED` 분개가 있으면 새 분개의 요청 상태와 관계없이 HTTP 409를 반환한다.
- 응답 코드는 기존 `APPROVED_JOURNAL_REPLACEMENT_BLOCKED`를 유지해 클라이언트와 검증 흐름의 의미를 일관되게 유지했다.
- 차단 메시지는 “비승인 분개 교체”에서 “새 분개 교체”로 확장했다.
- DB 워크플로우 검증에 승인 분개 생성 후 다시 `APPROVED` 분개로 교체하려는 시나리오를 추가했다.

## 실행 명령
- `npm run typecheck`
- `npm run lint`
- `git diff --check`
- `npm run verify:mvp`
- `if [ -n "$DATABASE_URL" ]; then printf 'DATABASE_URL=set\n'; else printf 'DATABASE_URL=missing\n'; fi`
