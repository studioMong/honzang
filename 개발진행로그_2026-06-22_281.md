# 개발 진행 로그 2026-06-22 281

## 작업 범위
- 운영 DB mutation workflow 검수를 임시 접근코드로 실행했다.
- 인증 읽기 검증은 재배포 반영 대기 후 통과했다.
- workflow 실행 중 검증 스크립트의 운영 DB 비교 가정과 거래 PATCH 라우트 버그를 발견했다.

## 발견 및 수정
- CSV template mapping 비교가 JSON 저장 과정에서 `undefined` 필드가 생략되는 점을 고려하지 못했다.
- CSV template mapping 문자열 비교가 key 순서에 영향을 받았다.
- 실패 시 import batch/template ID가 cleanup 목록에 늦게 들어가 일부 artifact가 남을 수 있었다.
- 존재하지 않는 계정코드 검증값이 길이 제한을 먼저 위반해 400이 발생했다.
- 거래 PATCH에서 `confirmedAccountId`를 보내지 않는 evidence-only 수정도 기존 confirmed account를 다시 검증해 404가 발생했다.

## 반영
- `scripts/verify-db-workflow.ts`를 운영 DB 직렬화와 cleanup 순서에 맞게 보강했다.
- `src/app/api/transactions/route.ts`에서 `confirmedAccountId`가 요청에 포함된 경우에만 계정 존재 검증을 하도록 수정했다.

## 운영 DB 정리
- 실패 run에서 남은 `verify-db-workflow-1782057405426` import batch 1개와 CSV template 1개를 삭제했다.
- 이후 `verify-db-workflow-*` import batch, template, classification rule, vendor, transaction 잔여값 0개를 확인했다.
