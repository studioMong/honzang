# 개발 진행 로그 222

## 작업 시각
- 2026-06-18

## 목표
- 혼자장부 MVP의 분개 저장/복원 경로에서 회계 금액 불변식을 더 엄격하게 보강한다.
- `Decimal(14,2)` DB 저장 한도와 맞지 않는 금액, 소수 자릿수 초과, 센트 단위 불균형 분개가 API 또는 백업 복원 경로로 들어오지 못하게 한다.

## 변경 내용
- `src/lib/server/money-validation.ts` 추가
  - `Decimal(14,2)` 금액 최대값 `999,999,999,999.99` 검증
  - 유한 숫자, 0 이상, 소수 둘째 자리 이하 검증
  - 금액을 1/100 단위로 변환해 합계 비교가 반올림 오차에 의존하지 않도록 처리
- `src/app/api/journals/route.ts`
  - 분개 라인의 차변/대변 금액에 공통 금액 검증 적용
  - 기존 `Math.round` 기반 균형 검사를 센트 단위 정확 비교로 변경
  - 차변/대변 불일치 시 `UNBALANCED_JOURNAL` 코드 반환
- `src/app/api/backups/restore/route.ts`
  - 백업 JSON 안의 분개 라인에도 같은 금액 검증 적용
  - 금액 오류가 있는 라인은 합계 계산에서 제외해 잘못된 숫자가 복원 검증 합계에 섞이지 않도록 정리
- `scripts/smoke-production.mjs`
  - 분개 금액 소수 자릿수 초과 거부 검증 추가
  - 센트 단위 차변/대변 불균형 거부 검증 추가
- `scripts/verify-backup-restore.mjs`
  - 백업 복원 dry-run에서 센트 단위 불균형 분개가 거부되는지 검증 추가
