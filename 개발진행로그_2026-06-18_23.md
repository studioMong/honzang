# 개발 진행 로그 - 2026-06-18 23

## 목표
- 새 API가 추가될 때 접근코드 보호 누락을 놓치지 않도록 접근제어 검증 범위를 넓힌다.
- CSV 템플릿 삭제, 백업 복원, 회사/리포트/증빙/분개 등 민감 API가 인증 없이 호출되지 않는지 자동 확인한다.

## 변경 내용
- `scripts/verify-access-control.mjs`
  - `verifyUnauthorizedApi`를 단일 API 확인에서 민감 API 목록 순회 방식으로 확장했다.
  - 다음 라우트가 접근 쿠키 없이 HTTP 401과 `AUTH_REQUIRED`를 반환하는지 확인한다.
    - `/api/companies`
    - `/api/transactions`
    - `/api/evidences`
    - `/api/journals`
    - `/api/reviews`
    - `/api/reports`
    - `/api/reports/summary`
    - `/api/vendors`
    - `/api/audit-events`
    - `/api/closing-periods`
    - `/api/operations/readiness`
    - `/api/classification-rules`
    - `/api/csv-templates`
    - `/api/backups/restore`

## 검증
- `npm run verify:access-control` 통과.
- `npm run lint` 통과.
- `git diff --check` 통과.
- `npm run verify:mvp` 통과.

## 판단
- 신규 API인 CSV 템플릿 삭제 라우트도 접근코드 보호 회귀 검증에 포함됐다.
- 운영 배포에서 민감 장부 API가 공개로 열리는 실수를 더 빨리 잡을 수 있다.
