# 개발 진행 로그 198

## 작업 주제
- 세무대리/자동신고로 오해될 수 있는 제품 문구 회귀 방지 검증 추가.

## 변경 사항
- README에 `제품 경계` 섹션을 추가했다.
  - 혼자장부는 대표가 직접 장부를 정리하고 신고 준비자료를 만드는 도구로 명시했다.
  - 세무 범위는 신고 준비자료 생성까지로 명시했다.
  - 홈택스 자동 제출, 신고 대행, 세무대리, 절세 자문, 법인세 신고서 자동 확정은 MVP 제외로 명시했다.
- `scripts/verify-product-boundaries.mjs`를 추가했다.
  - `src`와 `public`의 제품 노출 파일에서 신고 대행, 세무대리, 절세 자문, 자동 신고 등 금지 포지셔닝 문구가 없는지 검사한다.
  - README와 프로젝트 검토 문서가 제품 경계와 제외 범위를 유지하는지 검사한다.
  - 앱 화면/다운로드 payload에 직접 신고 준비, 홈택스/국세청 기준 최종 확인, 세무 전문가 검토 필요 문구가 유지되는지 검사한다.
- `package.json`에 `verify:product-boundaries`를 추가했다.
- `verify:mvp`에 `verify:product-boundaries` 단계를 추가했다.

## 검증
- `npm run verify:product-boundaries` 통과.
- `npm run typecheck` 통과.
- `npm run lint` 통과.
- `git diff --check` 통과.
- `npm run verify:mvp` 통과.

## DB 검증
- 현재 셸에 `DATABASE_URL`이 없어 `npm run verify:db-workflow`는 실행하지 않았다.

## 비고
- 이번 변경은 기능 추가보다 제품 경계 보강이다. 사용자가 직접 신고한다는 MVP 원칙이 UI와 다운로드 자료에서 흔들리지 않도록 자동 검증에 포함했다.
