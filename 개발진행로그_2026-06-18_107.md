# 개발 진행 로그 107

## 작업 단위
- 승인된 분개가 있는 거래의 계정과목 변경을 차단했다.
- 거래 원본의 확정 계정과 승인 분개/원장이 서로 달라지는 무결성 문제를 막는 서버 가드와 DB 워크플로우 회귀 검증을 추가했다.

## 변경 파일
- `src/app/api/transactions/route.ts`
- `scripts/verify-db-workflow.ts`

## 구현 내용
- 거래 `PATCH`에서 기존 거래일의 마감 기간 잠금을 확인한 뒤, `confirmedAccountId`가 실제로 변경되는 경우 승인된 분개 존재 여부를 조회한다.
- 같은 거래에 `APPROVED` 상태의 분개가 있으면 HTTP 409와 `APPROVED_JOURNAL_ACCOUNT_CHANGE_BLOCKED` 코드를 반환한다.
- 승인 분개가 있더라도 계정과목이 바뀌지 않는 요청, 증빙 상태 변경, 메모 변경은 기존처럼 허용한다.
- DB 워크플로우 검증에 승인 분개 생성 후 다른 계정과목으로 거래를 수정하려는 시나리오를 추가했다.

## 실행 명령
- `npm run typecheck`
- `npm run lint`
- `git diff --check`
- `npm run verify:mvp`
- `if [ -n "$DATABASE_URL" ]; then printf 'DATABASE_URL=set\n'; else printf 'DATABASE_URL=missing\n'; fi`
