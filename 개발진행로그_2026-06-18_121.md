# 개발 진행 로그 121

## 작업 단위
- 자동 분류 규칙 수정 시 존재하지 않는 계정과목 코드가 저장되지 않도록 서버 검증을 보강했다.

## 변경 파일
- `src/app/api/classification-rules/route.ts`
- `scripts/verify-db-workflow.ts`

## 구현 내용
- `PATCH /api/classification-rules`에서 `accountCode`가 포함된 경우 해당 회사의 활성 계정과목인지 확인한다.
- 계정과목을 찾을 수 없으면 HTTP 404와 `계정과목을 찾을 수 없습니다.` 메시지를 반환한다.
- DB 워크플로우 검증에 임시 자동 분류 규칙 생성 후 존재하지 않는 계정코드로 수정하려는 시나리오를 추가했다.
- 검증 중 생성된 임시 자동 분류 규칙은 cleanup 대상에 포함했다.

## 실행 명령
- `npm run typecheck`
- `npm run lint`
- `git diff --check`
- `npm run verify:mvp`
- `if [ -n "$DATABASE_URL" ]; then printf 'DATABASE_URL=set\n'; else printf 'DATABASE_URL=missing\n'; fi`

## 검증 결과
- TypeScript 타입체크 통과

- `verify:mvp` 통과
  - 배포 설정 검증 통과
  - 샘플 CSV 파싱/분류 검증 통과
  - production standalone build 통과
