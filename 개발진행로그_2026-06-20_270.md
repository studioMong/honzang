## 목표

- 운영 접근코드 보호가 켜진 Railway 환경에서도 `verify:db-workflow`를 실행할 수 있도록 검증 스크립트를 보강한다.

## 변경 사항

- `scripts/verify-db-workflow.ts`
  - `VERIFY_DB_WORKFLOW_ACCESS_CODE` 환경변수를 지원하도록 추가했다.
  - `VERIFY_DB_WORKFLOW_ACCESS_COOKIE` 환경변수도 지원해 이미 발급된 접근 쿠키로 검증할 수 있게 했다.
  - 실행 시작 시 `/api/auth/session`으로 접근 보호 여부를 확인한다.
  - 접근 보호가 켜져 있고 인증되지 않았으면 `VERIFY_DB_WORKFLOW_ACCESS_CODE`로 `/api/auth/login`을 호출해 HTTP-only 접근 쿠키를 받아 검증 요청에 사용한다.
  - 접근 보호가 켜져 있는데 코드/쿠키가 없으면 운영 데이터를 변경하기 전에 명확한 오류로 중단한다.
- `README.md`
  - 접근코드 보호 환경에서 `VERIFY_DB_WORKFLOW_ACCESS_CODE`를 함께 전달하는 예시를 추가했다.
- `docs/railway-cutover.md`
  - staging/production DB workflow 검증 예시에 `VERIFY_DB_WORKFLOW_ACCESS_CODE`를 반영했다.
- `scripts/verify-deployment-config.mjs`
  - cutover 문서에 접근코드 기반 DB workflow 안내가 있는지 검증하도록 추가했다.

## 메모

- 운영 DB workflow는 테스트 거래, 분개, 리포트, 마감 데이터를 만들고 정리하는 mutation 검증이다.
- 운영 접근 보호가 켜져 있으므로 실제 운영 DB workflow 실행에는 `VERIFY_DB_WORKFLOW_ACCESS_CODE` 값이 필요하다.
- 접근코드 없이 실행하면 데이터 변경 전에 중단되도록 했다.
