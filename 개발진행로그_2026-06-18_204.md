# 개발 진행 로그 204

## 작업 주제
- 워크스페이스 백업 export payload와 복원 dry-run API의 호환성 검증 추가.

## 변경 사항
- `scripts/verify-workspace-backup-restore-contract.ts`를 추가했다.
  - `buildWorkspaceBackupPayload()`가 생성한 실제 export JSON을 production standalone 서버의 `/api/backups/restore`에 dry-run으로 전송한다.
  - direct export payload와 ZIP 내부 `workspace-backup.json`이 모두 복원 dry-run을 통과하는지 검증한다.
  - bare export JSON을 그대로 복원 API에 보냈을 때 schema 검증을 통과하고 `RESTORE_CONFIRMATION_REQUIRED` 가드까지 도달하는지 확인한다.
  - 계정, CSV 템플릿, 가져오기 배치, 원본 CSV, 거래, 증빙, 분개, 리포트, 마감, 거래처, 자동분류, 감사 로그, 검토 항목 count가 복원 dry-run 응답과 일치하는지 검증한다.
- `package.json`에 `verify:workspace-backup-restore-contract`를 추가했다.
- `verify:mvp`에 `verify:workspace-backup-restore-contract` 단계를 추가했다.
  - production standalone 서버가 필요하므로 `build` 이후에 실행되도록 배치했다.

## 검증
- `npm run typecheck` 통과.
- `npm run verify:workspace-backup-restore-contract` 통과.
- `npm run lint` 통과.
- `git diff --check` 통과.
- `npm run verify:mvp` 통과.

## DB 검증
- 현재 셸에 `DATABASE_URL`이 없어 `npm run verify:db-workflow`는 실행하지 않았다.

## 비고
- 이번 변경으로 백업 다운로드 기능과 백업 복원 기능이 별도 검증에만 머무르지 않고, 같은 payload 계약으로 연결되었는지 자동 회귀 검증한다.
- Railway 공개 URL의 legacy/static 연결 문제는 이번 코드 변경과 별개로 계속 남아 있다.
