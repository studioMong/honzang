## 목표

- 설정 화면에서 거래처 기본값과 자동 분류 규칙을 삭제 후 재생성하지 않고 바로 수정할 수 있게 한다.
- 이미 존재하는 PATCH API를 화면에 연결하고, 접근제어/감사 로그 검증 계약을 보강한다.

## 변경

- `src/components/app-workspace.tsx`
  - 거래처 기본값 목록에 `수정` 버튼을 추가했다.
  - 선택한 거래처를 폼에 불러와 이름, 사업자등록번호, 기본 계정, 원천세 유형, 메모를 `수정 저장`할 수 있게 했다.
  - 거래처 수정 모드 취소 버튼을 추가했다.
  - 수정 중인 거래처를 삭제하면 폼을 초기화한다.
  - 자동 분류 규칙 목록에 `수정` 버튼을 추가했다.
  - 선택한 규칙을 폼에 불러와 규칙명, 키워드, 자료 유형, 계정과목, 우선순위를 `수정 저장`할 수 있게 했다.
  - 규칙 수정 모드 취소 버튼을 추가했다.
  - 수정 중인 규칙을 삭제하면 폼을 초기화한다.
- `scripts/verify-access-control.mjs`
  - `PATCH /api/vendors`, `PATCH /api/classification-rules`도 접근코드 없이 401이 되는지 확인하도록 추가했다.
- `scripts/verify-db-workflow.ts`
  - 기존 거래처/자동 분류 규칙 PATCH 흐름의 감사 로그가 각각 `VENDOR_UPDATE`, `CLASSIFICATION_RULE_UPDATE`로 남는지 확인하도록 추가했다.
- `scripts/smoke-production.mjs`
  - 설정 화면 스모크 검증에 `거래처 기본값`, `자동 분류 규칙` 섹션 노출을 추가했다.
- `README.md`
  - 거래처 기본 계정과 사용자 자동 분류 규칙을 추가/수정/삭제 가능한 기능으로 갱신했다.
