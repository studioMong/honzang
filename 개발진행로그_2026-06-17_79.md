# 개발진행로그 79

## 주제

접근코드 로그인 실패 제한과 배포 설정 검증 보강

## 변경 내용

- `src/lib/server/access-attempts.ts`를 추가해 접속 출처별 접근코드 실패 횟수를 서버 프로세스 메모리에 기록한다.
- 같은 접속 출처에서 접근코드를 5회 틀리면 10분간 로그인 요청을 `429`로 제한하고 `Retry-After` 헤더를 반환한다.
- 성공 로그인 시 해당 접속 출처의 실패 기록을 초기화한다.
- Railway 프록시 환경을 고려해 `x-forwarded-for`를 우선 접속 출처 키로 사용한다.
- `npm run verify:access-control`에 실패 횟수 제한, `Retry-After`, 잠금 중 정답 코드 차단 검증을 추가했다.
- `npm run verify:deployment-config`가 접근제어 검증 스크립트, proxy 보호 설정, README 환경변수 문서화를 확인하도록 확장했다.
- README에 접근코드 5회 실패 시 10분 제한 정책을 추가했다.
