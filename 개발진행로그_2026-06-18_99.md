# 개발 진행 로그 99

## 작업 단위
- Railway 운영 노출 전 민감 장부 앱에 필요한 기본 보안 응답 헤더를 전역 적용했다.

## 변경 내용
- `next.config.ts`
  - 모든 경로에 보안 헤더를 설정했다.
  - `Content-Security-Policy`: `base-uri 'self'`, `form-action 'self'`, `frame-ancestors 'none'`, `object-src 'none'`
  - `Cross-Origin-Opener-Policy`: `same-origin`
  - `Cross-Origin-Resource-Policy`: `same-origin`
  - `Permissions-Policy`: camera/geolocation/microphone/payment/usb 비활성화
  - `Referrer-Policy`: `no-referrer`
  - `Strict-Transport-Security`: 2년, includeSubDomains, preload
  - `X-Content-Type-Options`: `nosniff`
  - `X-Frame-Options`: `DENY`
- `scripts/verify-deployment-config.mjs`
  - Next 설정에 보안 헤더가 포함되어 있는지 회귀 검사한다.
- `scripts/smoke-production.mjs`
  - production standalone 서버의 `/`와 `/api/version` 응답에서 보안 헤더가 실제로 내려오는지 확인한다.

## 실행 명령
- `npm run verify:deployment-config`
- `npm run typecheck`
- `npm run lint`
- `git diff --check`
- `npm run verify:mvp`

## 검증 결과
- Deployment config verification passed.
- TypeScript typecheck 통과.
- ESLint 통과.
- Git whitespace 검사 통과.
- MVP verification passed.
- `smoke:prod`가 보안 헤더 응답 검증을 통과했다.

## 판단
- 운영 공개 도메인이 최신 Next.js 앱으로 연결되면 기본 브라우저 보안 헤더가 함께 적용된다.
- Next 런타임을 깨뜨릴 수 있는 광범위한 script CSP는 적용하지 않고, framing/object/base/form 범위부터 안전하게 제한했다.
