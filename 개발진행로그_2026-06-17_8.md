# 개발 진행 로그 - 2026-06-17 8차

## 이번 목표
- 기간별 신고 준비 리포트를 저장 가능한 마감 스냅샷으로 만든다.
- Railway Postgres 연결 시 `TaxReport` 테이블을 실제로 사용하게 한다.

## 구현 내용
- `AppTaxReport` 타입을 추가했다.
- `/api/reports` 라우트를 추가했다.
  - `GET`: 최근 저장된 세무 리포트 스냅샷을 조회한다.
  - `POST`: 선택 기간의 신고 준비 payload를 `TaxReport` 테이블에 저장한다.
  - DB가 없을 때는 샘플 모드 preview 응답으로 UI 흐름을 유지한다.
- `TaxReport` serializer를 추가했다.
- 리포트 화면에 `스냅샷 저장` 버튼을 추가했다.
  - 선택된 기간, 요약, 신고 패키지, 원천세 후보, 법인세 체크, 원장 행 수를 payload로 저장한다.
- 리포트 화면에 `저장된 리포트` 이력 표를 추가했다.
  - 저장일, 기간, 유형, 거래 건수, 손익, 예상 부가세를 확인할 수 있다.

## 검증
- `npm run lint` 통과.
- `npm run typecheck` 통과.
- `npm run build` 통과.
- 로컬 API 확인:
  - `GET /api/reports` 샘플 응답 확인.
  - `POST /api/reports` 샘플 저장 응답 확인.
- 로컬 브라우저 확인:
  - `/?view=reports`에서 `스냅샷 저장` 버튼 확인.
  - 버튼 클릭 후 `저장된 리포트` 섹션이 표시되는 것 확인.
  - 샘플 저장 이력에서 `법인세 준비`, `5건`, 손익, 예상 부가세가 표시되는 것 확인.

## 다음 우선순위
- 실제 Railway Postgres 연결 후 `/api/reports` 저장/조회 검증이 필요하다.
