# 개발 진행 로그 212

## 작업 주제
- 쿠키 기반 접근 보호 API에 cross-origin mutation 차단을 추가했다.

## 구현 내용
- `src/proxy.ts`에서 `POST`, `PUT`, `PATCH`, `DELETE` API 요청의 `Origin`을 검사하도록 했다.
- `Origin` 헤더가 없으면 기존 서버 간 호출과 smoke 검증 호환성을 위해 통과한다.
- `Origin` 헤더가 있고 앱 origin과 다르면 `403 INVALID_ORIGIN`으로 거부한다.
- 운영 프록시와 로컬 standalone 서버의 host 해석 차이를 고려해 `nextUrl`, `Host`, `X-Forwarded-Host`, `X-Forwarded-Proto` 기준 origin을 함께 인정한다.
- 공개 로그인 API와 인증된 mutation API 모두 같은 origin 정책을 적용한다.
- README의 접근 보호 설명에 API 변경 요청의 origin 검증을 반영했다.

## 검증 보강
- `scripts/verify-access-control.mjs`에 다음 검증을 추가했다.
  - cross-origin `/api/auth/login` 요청이 `403 INVALID_ORIGIN`으로 거부되는지 확인
  - 인증 쿠키가 있어도 cross-origin `/api/transactions` mutation이 route 처리 전에 `403 INVALID_ORIGIN`으로 거부되는지 확인
  - same-origin logout은 정상 처리되는지 확인
- `scripts/verify-deployment-config.mjs`에서 프록시의 `INVALID_ORIGIN`, `MUTATION_METHODS` 계약을 확인하도록 했다.
