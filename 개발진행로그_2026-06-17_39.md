# 개발 진행 로그 39

## 목표

- 웹/앱 지원 요구에 맞춰 PWA 기본 동작이 배포 전 자동으로 검증되게 만든다.

## 작업 내용

- `scripts/verify-pwa.mjs`를 추가했다.
- `npm run verify:pwa` 명령을 추가했다.
- 검증 스크립트는 production standalone 서버를 실행한 뒤 다음 항목을 확인한다.
  - `/manifest.webmanifest` 200 응답
  - 앱 이름, `standalone` 표시 모드, `start_url`, `scope`, 테마 컬러
  - 아이콘과 maskable 아이콘
  - CSV 업로드, 리포트 바로가기
  - `/sw.js` 200 응답
  - service worker의 install/fetch 이벤트와 오프라인 fallback
  - `/offline.html` 200 응답과 오프라인 안내 문구
  - `/icon.svg` 200 응답
  - 앱 소스의 `navigator.serviceWorker.register("/sw.js")` 등록 코드
- README에 `npm run build` 후 `npm run verify:pwa`로 PWA 리소스를 검증하는 절차를 추가했다.

## 검증

- `npm run typecheck` 통과
- `npm run lint` 통과
- `npm run build` 통과
- `npm run verify:pwa` 통과
- `npm run verify:samples` 통과
- `npm run smoke:prod` 통과

## 비고

- 이번 작업은 PWA 기능 자체를 크게 바꾸기보다, 설치형 앱 지원이 빠지지 않도록 검증 게이트를 추가한 것이다.
- 네이티브 앱 래핑은 아직 진행하지 않았고, PWA 안정화 후 Capacitor 래핑을 검토하는 흐름을 유지한다.
