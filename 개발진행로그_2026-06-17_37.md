# 개발 진행 로그 37

## 목표

- 신고 준비 ZIP 패키지에 DB에 보관된 소형 증빙 원본 파일도 함께 포함한다.

## 작업 내용

- `패키지 ZIP` 생성 시 `filteredEvidences`를 함께 넘기도록 변경했다.
- `fileDataUrl`이 있는 증빙을 ZIP의 `evidences/` 폴더에 추가한다.
- `manifest.json`에 `evidenceFiles` 목록을 포함해 ZIP 안의 증빙 파일 위치를 확인할 수 있게 했다.
- 증빙 파일명은 다음 기준으로 정리한다.
  - 브라우저 경로가 섞인 파일명은 마지막 파일명만 사용
  - ZIP 경로에 위험한 문자와 제어문자는 `_`로 치환
  - 파일명이 비어 있으면 MIME 타입 기준 확장자를 붙인 `evidence-N` 이름 사용
  - 같은 파일명이 여러 개 있으면 `-2`, `-3`처럼 번호를 붙임
- README의 MVP 범위를 `신고 준비자료 JSON/CSV/DB 보관 증빙 ZIP 패키지 다운로드`로 갱신했다.

## 검증

- `npm run typecheck` 통과
- `npm run lint` 통과
- `npm run verify:samples` 통과
- `npm run build` 통과
- `npm run smoke:prod` 통과
- 로컬 production 서버 리포트 화면 SSR 확인
  - `/?view=reports` 200 응답
  - `패키지 ZIP` 문구 확인
  - `신고 준비 리포트` 문구 확인

## 비고

- 이번 확장은 이미 DB에 보관된 750KB 이하 증빙 파일만 대상으로 한다.
- 대용량 증빙 원본까지 포함하려면 오브젝트 스토리지 도입 후 ZIP 생성 방식을 다시 설계하는 편이 낫다.
