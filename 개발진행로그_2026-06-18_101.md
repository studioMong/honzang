# 개발 진행 로그 101

## 작업 단위
- production 보안 헤더 검증을 로컬 smoke뿐 아니라 Railway 공개 URL 검증/감사에도 포함했다.

## 변경 내용
- `scripts/lib/security-headers.mjs`
  - 보안 헤더 기대값을 공용 검사 모듈로 분리했다.
  - CSP, COOP, CORP, Permissions-Policy, Referrer-Policy, HSTS, nosniff, X-Frame-Options 누락을 검사한다.
- `scripts/smoke-production.mjs`
  - 중복된 헤더 검사 배열을 공용 `findSecurityHeaderIssues()` 사용으로 교체했다.
- `scripts/verify-railway.mjs`
  - 공개 URL의 `/`와 `/api/version` 응답에 production 보안 헤더가 있는지 필수 검증한다.
  - 실패 diagnostics에 보안 헤더 누락 내용을 출력한다.
- `scripts/audit-railway-deployment.mjs`
  - 공개 URL 감사 결과에 `보안 헤더 누락` fail finding 또는 `보안 헤더 응답` pass finding을 추가한다.
- `scripts/verify-deployment-config.mjs`
  - 공용 헤더 검사 모듈과 Railway 검증/감사의 보안 헤더 연결을 회귀 검사한다.

## 실행 명령
- `npm run verify:deployment-config`
- `npm run typecheck`
- `npm run lint`
- `git diff --check`
- `npm run verify:mvp`

## 판단
- Railway public domain이 최신 Next.js 앱으로 전환된 뒤에는 API/DB/PWA뿐 아니라 보안 헤더까지 배포 성공 조건에 포함된다.
- 현재 legacy/static 공개 URL은 보안 헤더 누락도 별도 감사 항목으로 드러나게 된다.
