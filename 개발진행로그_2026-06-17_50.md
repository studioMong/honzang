# 개발진행로그 50 - DB 워크플로우 월 마감 검증 보강

## 진행 내용

- `scripts/verify-db-workflow.ts`에 월 마감 잠금 회귀 검증을 추가했다.
- DB 워크플로우 검증 중 리포트 스냅샷 저장 후 `2026-06` 기간을 마감 잠금한다.
- 잠긴 월의 수기 거래 생성이 `409 PERIOD_CLOSED`로 차단되는지 확인한다.
- 잠긴 월의 리포트 삭제가 `409 PERIOD_CLOSED`로 차단되는지 확인한다.
- 마감 해제 후 감사 로그에 `PERIOD_CLOSE`, `PERIOD_REOPEN`이 남는지 확인한다.
- 실패 시에도 정리 단계에서 마감 잠금을 먼저 해제하도록 cleanup 순서를 보강했다.
- README의 DB 워크플로우 검증 설명에 월 마감 잠금 범위를 반영했다.

## 검증

- `npm run typecheck` 통과
- `npm run lint` 통과

## 참고

- `npm run verify:db-workflow`는 실제 DB 모드 앱 서버가 필요하므로 로컬/스테이징 DB가 연결된 서버에서 실행해야 한다.
