# 개발진행로그 80

## 주제

접근코드 실패 제한 커밋 후 Railway 공개 URL 재진단

## Git 상태

- 커밋: `2de048af18a2d921227ade65a0261671992cfe53`
- 메시지: `Throttle access code attempts`
- 원격: `origin/main` 푸시 완료

## Railway 확인 결과

```bash
RAILWAY_EXPECTED_COMMIT=2de048af18a2d921227ade65a0261671992cfe53 npm run verify:railway
```

결과:

- 실패
- `/`: HTTP 200, legacy 정적 프로젝트 요약 페이지
- `/api/version`: HTTP 404
- `/api/health`: HTTP 404
- `/manifest.webmanifest`: HTTP 404

```bash
RAILWAY_AUDIT_SOFT=1 RAILWAY_EXPECTED_COMMIT=2de048af18a2d921227ade65a0261671992cfe53 npm run audit:railway
```

결과:

- `4 fail, 0 warn, 0 pass`
- Next.js API 라우트 미노출
- legacy 정적 페이지 응답
- healthcheck 미노출
- PWA manifest 미노출

## 판단

새 커밋을 GitHub 원격에 푸시했지만 공개 URL은 여전히 최신 Next.js standalone 서버를 제공하지 않는다. 동일하게 이전 정적/legacy 서비스가 응답하고 있으므로 Railway Dashboard에서 public domain 연결 서비스와 최신 Deploy source commit을 확인해야 한다.
