# 개발진행로그 26

## 목표

반복 거래처의 기본 계정과 원천세 확인 유형을 저장해 CSV import와 수기 거래 분류 정확도를 높인다.

## 변경 내용

- `AppVendor` 타입을 추가했다.
- `Vendor` 직렬화 함수를 추가했다.
- `/api/vendors`를 추가해 거래처 조회, 생성, 수정, 삭제를 지원한다.
- 설정 화면에 `거래처 기본값` 섹션을 추가했다.
- 거래처명, 사업자등록번호, 기본 계정, 원천세 유형, 메모를 입력할 수 있게 했다.
- CSV import 저장 시 거래처 기본 계정을 먼저 적용하고, 사용자 자동 분류 규칙이 이후에 덮어쓸 수 있게 했다.
- 수기 거래 생성 시 DB 모드에서 거래처 기본 계정을 추천 계정에 반영한다.
- DB 거래 직렬화에서 `rawPayload.reviewReasons`를 복원하도록 했다.

## 검증 결과

- `npm run typecheck` 통과
- `npm run lint` 통과
- `npm run verify:samples` 통과
- `npm run build` 통과
- `npm run smoke:prod` 통과
- `/api/vendors` sample mode 조회/생성 응답 확인
- 설정 화면 거래처 섹션 렌더링 확인
- production smoke에 `/api/vendors` 확인 추가
