# 개발 진행 로그 83

## 작업 단위
- 증빙 생성/삭제 시 거래의 증빙상태가 실제 금액 대조 결과를 반영하도록 개선했다.

## 변경 내용
- `src/lib/server/evidence-amount-reviews.ts`
  - 거래와 연결 증빙 합계로 증빙상태를 계산하는 `resolveTransactionEvidenceStatus`를 추가했다.
  - 연결 증빙이 없으면 출금 거래는 `MISSING`, 입금 거래는 `UNCHECKED`로 계산한다.
  - 연결 증빙 합계가 거래금액과 일치하면 `MATCHED`, 증빙은 있으나 금액이 맞지 않으면 `ATTACHED`로 계산한다.
- `src/app/api/evidences/route.ts`
  - 증빙 생성 후 해당 거래의 모든 연결 증빙을 다시 읽어 증빙상태를 재계산한다.
  - 증빙 삭제 후 남은 증빙이 있어도 합계 기준으로 `MATCHED/ATTACHED/MISSING/UNCHECKED`를 다시 계산한다.
- `scripts/verify-sample-csvs.ts`
  - 증빙상태 계산의 주요 케이스를 회귀 검증에 추가했다.
