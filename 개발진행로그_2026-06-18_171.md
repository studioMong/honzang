## 구현 내용
- `src/app/api/transactions/route.ts`의 수기 거래 생성과 거래 수정 요청에 공통 JSON 파서(`parseJsonRequest`)를 적용했다.
- `src/app/api/journals/route.ts`의 분개 저장과 분개 상태 변경 요청에 공통 JSON 파서를 적용했다.
- `src/app/api/evidences/route.ts`의 증빙 추가와 삭제 요청에 공통 JSON 파서를 적용했다.
- 증빙 추가 요청은 파일 data URL을 포함할 수 있으므로 `MAX_EVIDENCE_FILE_DATA_URL_LENGTH + 20_000` 바이트까지 허용하도록 별도 한도를 뒀다.
- 잘못된 JSON은 `INVALID_JSON_PAYLOAD`와 HTTP 400으로 응답한다.
- `scripts/smoke-production.mjs`에 거래, 분개, 증빙 API의 malformed JSON 거부 검증을 추가했다.
