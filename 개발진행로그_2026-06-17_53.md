# 개발진행로그 53 - 리포트 화면 부가세 정리표 노출

## 진행 내용

- 리포트 화면에 `부가세 입력 전 정리표` 섹션을 추가했다.
- 기존 신고 패키지/CSV/XLSX에 들어가던 부가세 준비 행을 화면에서도 그대로 재사용한다.
- 표에는 다음 항목을 표시한다.
  - 구분
  - 신고서 입력/확인 위치
  - 공급가액
  - 세액
  - 건수
  - 거래/근거
  - 검토 메모
- `부가세` 다운로드 버튼을 정리표 섹션에도 배치해 화면 확인 후 바로 CSV를 내려받을 수 있게 했다.

## 검증

- `npm run typecheck` 통과
- `npm run lint` 통과
- `npm run build` 통과
- `npm run smoke:prod` 통과
- Browser 데스크톱 검증
  - `/?view=reports` 프로덕션 화면에서 `부가세 입력 전 정리표` 렌더링 확인
  - 9개 행 렌더링 확인
  - 다운로드 버튼 렌더링 확인
  - 콘솔 오류 없음
- Browser 모바일 폭 390px 검증
  - 페이지 전체 가로 오버플로우 없음
  - 표 내부 가로 스크롤 동작 구조 확인
  - 콘솔 오류 없음

## 참고

- 이 표는 홈택스 입력 전 검토용이다. 최종 신고 전에는 증빙 원본, 홈택스 자료, 국세청 공지를 확인해야 한다.
- 커밋 푸시 후 `RAILWAY_EXPECTED_COMMIT=0173a58 npm run verify:railway`를 실행했지만 공개 URL은 여전히 기존 정적 프로젝트 요약 페이지를 서빙하고 `/api/version`, `/api/health`, `/manifest.webmanifest`는 404로 응답했다.
