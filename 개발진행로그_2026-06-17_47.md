# 개발 진행 로그 47

## 목표

- 세무/장부 작업의 주요 변경 이력을 남겨 나중에 신고 자료와 원장 변경 과정을 설명할 수 있게 한다.

## 작업 내용

- Prisma `AuditEvent` 모델과 마이그레이션을 추가했다.
- `GET /api/audit-events` API를 추가했다.
- 설정 화면에 `활동 로그` 패널을 추가했다.
- 전체 백업 JSON에 감사 이력 `auditEvents`를 포함하도록 했다.
- 백업 복원 시 백업에 포함된 감사 이력을 복원하고, 복원 작업 자체도 `BACKUP_RESTORE` 이벤트로 남기도록 했다.
- 다음 주요 쓰기 API에 감사 이력 기록을 연결했다.
  - 회사 설정 저장
  - CSV 가져오기
  - 업로드 배치 삭제
  - 수기 거래 추가
  - 거래 수정
  - 증빙 추가
  - 분개 저장/승인
  - 분개 상태 변경
  - 리포트 스냅샷 저장/삭제
  - 거래처 기본값 추가/수정/삭제
  - 자동 분류 규칙 추가/수정/삭제
  - 검토 항목 상태 변경
  - 백업 복원
- `smoke:prod`가 `/api/audit-events` 응답을 확인하도록 했다.
- `verify:backup-restore`가 설정 화면의 활동 로그 섹션과 백업 감사 이력 dry-run 개수를 확인하도록 했다.
- `verify:db-workflow`가 실제 DB 흐름에서 `IMPORT_CREATE`, `JOURNAL_CREATE`, `REPORT_CREATE` 이벤트가 생성되는지 확인하도록 했다.
- README의 MVP 범위에 주요 작업 활동 로그를 추가했다.

## 검증

- `npm run typecheck` 통과
- `npm run lint` 통과
- `npm run build` 통과
- `npm run verify:backup-restore` 통과
- `npm run verify:samples` 통과
- `npm run smoke:prod` 통과
- `npm run verify:pwa` 통과
- `npm run verify:db-workflow`는 로컬 또는 스테이징 Postgres가 필요한 실제 DB 쓰기 검증이라 이번 환경에서는 실행하지 않았다.
