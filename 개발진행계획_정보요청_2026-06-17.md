# 혼자장부 개발 진행 계획 및 정보 요청

작성일: 2026-06-17

## 1. 진행 원칙

혼자장부는 세무대리 서비스가 아니라 `1인법인 대표가 직접 장부를 정리하고 신고 준비자료를 만드는 회계/장부 관리 웹앱`으로 개발한다.

초기 목표는 홈택스 자동 제출이 아니라 `CSV 업로드 기반 신고 준비 패키지 생성`이다.

## 2. 개발 진행 순서

### 1단계: 제품 기준 확정

목표:
- MVP 범위 확정
- 제외 범위 확정
- 기본 사용자 시나리오 확정
- 세무대리 리스크를 피하는 문구/기능 경계 확정

산출물:
- 제품 요구사항 문서
- MVP 기능 목록
- 사용자 플로우
- 화면 목록

### 2단계: 기술 스택 및 실행 방식 확정

목표:
- 로컬 앱으로 시작할지, 웹앱으로 시작할지 결정
- 프론트엔드/백엔드/DB 구성 결정
- 파일 저장 방식 결정

기본 추천:
- 초기: 로컬 실행 가능한 웹앱
- 프론트엔드: React + TypeScript
- 백엔드: Node.js 또는 Next.js API
- DB: SQLite 또는 PostgreSQL
- 파일: 원본 CSV 보관 + 파싱 결과 DB 저장

### 3단계: 데이터 모델 설계

목표:
- 회사 정보
- 계정과목
- 거래 원본
- 증빙
- 분개
- 검토 항목
- 세금 리포트
- 자동 분류 규칙

핵심 원칙:
- 원본 CSV는 덮어쓰지 않는다.
- 앱 추천값과 사용자 확정값을 분리한다.
- 모든 자동 판단은 `추천`, `검토 필요`, `주의`로 표현한다.

### 4단계: CSV 업로드 MVP

목표:
- 통장 CSV 업로드
- 카드 CSV 업로드
- 파싱 결과 미리보기
- 원본 파일 보관
- 거래 목록 생성

첫 구현 우선순위:
1. 회사 설정
2. 통장 CSV 업로드
3. 카드 CSV 업로드
4. 거래 목록
5. 계정과목 수동 지정

### 5단계: 거래 분류와 규칙

목표:
- 기본 계정과목 세트 구성
- 거래처/적요 기반 자동 분류
- 사용자가 수정하면 규칙으로 저장
- 미분류 거래 검토함 생성

### 6단계: 증빙과 검토함

목표:
- 증빙 파일 첨부
- 증빙 상태 표시
- 증빙 없는 비용 경고
- 대표자 거래, 개인 사용 의심 거래, 원천세 검토 항목 표시

### 7단계: 리포트

목표:
- 월별 손익 리포트
- 부가세 집계
- 원천세 체크리스트
- 법인세 결산 준비 리포트
- 계정별 원장 초안

### 8단계: 검증 및 사용성 개선

목표:
- 샘플 CSV로 반복 테스트
- 실제 Studio Mong 흐름에 맞게 계정과목/규칙 보정
- 오류 처리
- 모바일/데스크톱 화면 검증

### 9단계: 내부 사용 버전 완성

목표:
- 한 회사 기준으로 월별 장부 정리 가능
- 신고 전 숫자 확인 가능
- 증빙 누락과 위험 거래 확인 가능
- PDF/CSV/엑셀 형태의 신고 준비자료 다운로드 가능

## 3. 사용자가 제공해야 하는 정보

### A. 반드시 필요한 정보

1. 초기 사용 형태
   - 내부용만 쓸지
   - 나중에 외부 SaaS까지 고려할지

2. 실행 방식
   - 내 컴퓨터에서만 쓰는 로컬 웹앱
   - 로그인 가능한 웹 서비스
   - 둘 다 가능하게 시작

3. 회사 기본 정보
   - 법인명
   - 업종
   - 결산월
   - 과세유형
   - 대표자 급여 지급 여부
   - 직원/외주 지급 여부

4. 실제 거래 자료 샘플
   - 법인통장 CSV 샘플
   - 법인카드 CSV 샘플
   - 홈택스 전자세금계산서 CSV 샘플
   - 홈택스 현금영수증/카드 매입자료 샘플
   - PG 정산자료가 있으면 샘플

주의:
- 실제 계좌번호, 사업자번호, 거래처 정보가 부담되면 일부 마스킹해서 줘도 된다.
- 샘플은 컬럼 구조가 실제와 같아야 한다.

5. 계정과목 운영 방식
   - 아주 단순한 1인법인용 계정과목으로 시작
   - 표준 계정과목에 가깝게 시작
   - 기존 세무사가 쓰던 계정과목이 있으면 그 기준 사용

### B. 있으면 개발 속도가 빨라지는 정보

1. 월 평균 거래량
   - 통장 입출금 건수
   - 카드 사용 건수
   - 매출 건수
   - 외주/급여 지급 건수

2. 주요 거래 유형
   - 광고비
   - 소프트웨어/SaaS
   - 외주비
   - 인건비
   - 접대/식대
   - 해외 결제
   - 대표자 입출금

3. 신고 방식
   - 현재 세무사를 쓰는지
   - 직접 홈택스 신고 경험이 있는지
   - 부가세/원천세/법인세 중 어디까지 직접 하고 싶은지

4. 출력물 요구
   - CSV 다운로드
   - 엑셀 다운로드
   - PDF 리포트
   - 홈택스 입력값 형태의 요약표

5. 디자인 방향
   - 조용한 업무용 대시보드
   - 엑셀 같은 밀도 높은 화면
   - 토스/세무 앱처럼 쉬운 화면

## 4. 내가 기본값으로 정할 수 있는 항목

사용자가 별도 선호를 주지 않으면 다음 기준으로 진행한다.

- 제품 형태: 로컬 우선 웹앱
- 대상: 1인법인 1개 회사
- 초기 데이터: CSV 수동 업로드
- DB: SQLite
- 화면: 대시보드, 회사설정, 업로드, 거래내역, 검토함, 리포트, 설정
- 세무 범위: 신고 준비자료 생성까지
- 신고 제출: 제외
- 자동 판단 표현: 추천/주의/검토 필요
- 디자인: 조용하고 밀도 있는 업무용 UI

## 5. 다음 사용자 응답 형식

아래 항목만 먼저 답하면 개발 착수가 가능하다.

1. 내부용만 먼저 만들지, SaaS까지 고려해서 구조를 잡을지
2. 로컬 웹앱으로 시작해도 되는지
3. 사용할 기술 스택 선호가 있는지
4. 실제 CSV 샘플을 제공할 수 있는지
5. 법인 기본 정보와 대표자 급여/외주 지급 여부
6. 첫 버전에서 꼭 나와야 하는 리포트 3개

이 정보가 정해지면 MVP 개발을 시작한다.
