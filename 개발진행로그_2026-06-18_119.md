# 개발 진행 로그 119

## 작업 단위
- 설정 화면의 회사 설정, 거래처 기본값, 자동 분류 규칙, CSV 매핑 템플릿 변경 실패 처리를 보강했다.

## 변경 파일
- `src/components/app-workspace.tsx`

## 구현 내용
- 설정 화면 상단에 공통 `settingsMessage`를 추가했다.
- 회사 설정 저장은 `response.ok`를 확인한 뒤에만 저장 완료 상태를 반영하고, 실패 시 오류 메시지를 표시한다.
- 거래처 기본값 추가/삭제 실패 시 사용자에게 이유를 표시하고 삭제 실패는 이전 목록으로 롤백한다.
- 자동 분류 규칙 추가, 사용/중지, 삭제 실패 시 메시지를 표시하고 낙관적 변경을 롤백한다.
- CSV 매핑 템플릿 삭제 실패 시 이전 목록으로 되돌리고 실패 메시지를 표시한다.

## 실행 명령
- `npm run typecheck`
- `npm run lint`
- `git diff --check`
- `npm run verify:mvp`
- `if [ -n "$DATABASE_URL" ]; then printf 'DATABASE_URL=set\n'; else printf 'DATABASE_URL=missing\n'; fi`
