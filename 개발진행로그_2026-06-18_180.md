# 개발 진행 로그 180

## 작업 주제
- 자료 수집 현황의 기간별 업로드 배치 집계 보정
- 자료 수집 상태 전용 검증 스크립트 추가

## 구현 내용
- `src/lib/data-sources.ts`
  - 거래에 `importBatchId`가 있으면 해당 거래와 연결된 업로드 배치만 집계하도록 보정했다.
  - 선택 월/리포트 기간에서 과거 업로드 배치가 현재 기간의 업로드 수로 섞이는 문제를 줄였다.
  - 거래에 `importBatchId`가 없는 경우에는 기존처럼 출처별 업로드 이력을 표시하도록 fallback을 유지했다.
- `scripts/verify-data-sources.ts`
  - DB 없이 실행되는 자료 수집 상태 검증을 추가했다.
  - 통장 거래가 현재 기간 배치 1개만 세는지 확인했다.
  - 원본 CSV가 없는 카드 업로드 이력은 `원본 CSV 0/1개`로 표시되는지 확인했다.
  - 미업로드 필수 자료와 선택 자료인 PG 상태를 확인했다.
- `package.json`
  - `verify:data-sources` 스크립트를 추가했다.
- `scripts/verify-mvp.mjs`
  - 전체 MVP 검증에 `verify:data-sources` 단계를 포함했다.

## 검증
- `npm run verify:data-sources`
  - 최초 샌드박스 실행은 `tsx` IPC 파이프 권한 문제로 실패
  - 권한 상승 재실행 통과
- `npm run typecheck` 통과
- `npm run lint` 통과
- `git diff --check` 통과
- `npm run verify:mvp` 통과
- 브라우저 확인 통과
  - `http://127.0.0.1:3125/`
  - 대시보드 `자료 수집 현황`, `업로드 이력`, `원본 CSV` 표시 확인
  - 리포트 탭 `업로드`, `원본` 열 표시 확인
  - 브라우저 콘솔 오류 0건
