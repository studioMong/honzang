# 개발진행로그 11 - 빈 DB 초기 시작 UX 보강

## 목표

Railway Postgres가 새로 연결된 직후 거래가 0건인 상태에서도 사용자가 다음 작업 순서를 바로 볼 수 있게 한다.

## 구현 내용

- 대시보드에서 거래가 0건이면 `초기 장부 시작` 패널을 표시한다.
- 초기 단계는 다음 순서로 구성했다.
  - 법인 기본정보 확정
  - 법인 통장 CSV 업로드
  - 카드·홈택스 자료 반영
  - CSV에 없는 거래 보완
  - 분개 승인 후 리포트 저장
- 각 단계는 실제 화면으로 이동하는 액션 버튼을 포함한다.
- 최근 거래 공용 표와 거래내역 편집 표에 거래 0건 안내 행을 추가했다.
- 샘플 데이터 모드처럼 거래가 이미 있는 경우에는 초기 시작 패널을 숨긴다.

## 검증

- `npm run typecheck` 통과
- `npm run lint` 통과
- `npm run verify:mvp` 통과
  - Railway/standalone 배포 설정 검증
  - TypeScript 타입체크
  - ESLint
  - CSV 샘플 파싱과 자동분류 검증
  - 프로덕션 standalone 빌드
  - PWA 검증
  - 접근코드 보호 검증
  - 백업 복원 dry-run
  - production API/UI smoke
- 브라우저 확인
  - 샘플 모드 데스크톱 1280px에서 대시보드와 거래내역 정상 표시
  - 거래가 있는 샘플 모드에서는 `초기 장부 시작` 패널이 숨겨짐
  - 모바일 390px에서 대시보드와 거래내역 문서 전체 가로 overflow 없음

## 남은 외부 이슈

- Railway 공개 URL은 여전히 Dashboard 설정 확인이 필요하다.
- 외부 도메인이 최신 Next.js 서비스로 연결되면 빈 DB 상태에서 초기 시작 패널이 실제 운영 화면에도 표시되는지 최종 확인한다.
