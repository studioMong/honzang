# 개발 진행 로그 41

## 목표

- Railway 공개 URL이 최신 Next 서버와 Railway Postgres를 바라보는지 자동으로 판정할 수 있게 만든다.

## 작업 내용

- `scripts/verify-railway.mjs`를 추가했다.
- `npm run verify:railway` 명령을 추가했다.
- 검증 항목은 다음과 같다.
  - `/api/version`이 `honzang` 앱 이름과 production 환경을 반환하는지
  - Railway 메타데이터 `RAILWAY_GIT_COMMIT_SHA`, `RAILWAY_SERVICE_NAME`이 노출되는지
  - `RAILWAY_EXPECTED_COMMIT`을 지정하면 공개 배포 커밋이 일치하는지
  - `/api/health`가 `ok: true`, `database: connected`를 반환하는지
  - 루트 페이지가 `혼자장부` 앱이며 예전 `프로젝트 정리` 정적 HTML이 아닌지
  - `manifest.webmanifest`가 설치형 PWA 기본값을 반환하는지
- README에 `npm run verify:railway`와 커밋 고정 검증 예시를 추가했다.

## 검증

- `npm run typecheck` 통과
- `npm run lint` 통과
- `npm run build` 통과
- 로컬 standalone 대상 `verify:railway` 통과
  - `RAILWAY_PUBLIC_URL=http://127.0.0.1:3103 RAILWAY_VERIFY_ALLOW_NO_DB=1 npm run verify:railway`
- `npm run smoke:prod` 통과
- `npm run verify:pwa` 통과
- `npm run verify:samples` 통과
- 실제 공개 URL 대상 `npm run verify:railway` 실행 결과
  - `/api/version` 404로 실패
  - 현재 `https://honzang-production.up.railway.app` 도메인이 최신 Next 앱 서비스에 연결되어 있지 않은 상태로 판단한다.

## 다음 조치

- Railway 대시보드에서 공개 도메인 `honzang-production.up.railway.app`이 현재 GitHub `studioMong/honzang` `main` 브랜치 서비스에 연결되어 있는지 확인한다.
- 최신 배포 커밋이 이번 커밋 이상인지 확인한다.
- `DATABASE_URL`이 Postgres 서비스 참조 변수로 연결되어 있는지 확인한다.
- 도메인 연결 수정 후 `RAILWAY_EXPECTED_COMMIT=$(git rev-parse HEAD) npm run verify:railway`가 통과해야 공개 배포 완료로 본다.
