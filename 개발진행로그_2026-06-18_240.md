# 개발진행로그 2026-06-18 240

## 목표

- 증빙을 삭제 후 다시 올리지 않고 증빙함에서 메타데이터와 연결 거래를 수정할 수 있게 한다.
- 증빙 수정도 월 마감 잠금, 연결 거래 증빙 상태 재계산, 감사 로그 흐름을 우회하지 못하게 한다.

## 변경

- `src/app/api/evidences/route.ts`
  - `PATCH /api/evidences`를 추가했다.
  - 증빙 유형, 발행일, 거래처, 사업자등록번호, 공급가액, 부가세, 총액, 연결 거래를 수정할 수 있게 했다.
  - 기존 발행일/변경 발행일, 기존 연결 거래일/변경 연결 거래일이 마감된 기간이면 `PERIOD_CLOSED`로 차단한다.
  - 금액은 공급가액+부가세와 총액 일관성 검증을 다시 통과해야 한다.
  - 연결 거래가 바뀌면 이전 거래와 새 거래의 증빙 상태를 재계산한다.
  - `EVIDENCE_UPDATE` 감사 로그를 남긴다.
- `src/components/app-workspace.tsx`
  - 증빙 목록에 `수정` 버튼을 추가했다.
  - 선택한 증빙을 폼에 불러와 `수정 저장`할 수 있게 했다.
  - 수정 모드에서는 파일 교체 없이 메타데이터와 연결 거래만 바꾼다.
  - 수정 중인 증빙을 삭제하면 편집 폼을 초기화한다.
  - 감사 로그 라벨에 `증빙 수정`을 추가했다.
- `scripts/verify-db-workflow.ts`
  - 증빙 생성 후 불일치 검토가 생기는지 확인한다.
  - 같은 증빙을 다른 거래로 이동하고 금액을 맞춰 이전 거래는 `UNCHECKED`, 새 거래는 `MATCHED`로 재계산되는지 검증한다.
  - 마감 후 증빙 PATCH가 `PERIOD_CLOSED`로 차단되는지 검증한다.
  - 감사 로그에 `EVIDENCE_UPDATE`가 남는지 검증한다.
- `scripts/verify-period-locks.mjs`
  - DB 워크플로가 잠긴 기간의 증빙 수정 차단을 검증하도록 계약을 추가했다.
- `scripts/verify-access-control.mjs`
  - 접근코드 보호 모드에서 `PATCH /api/evidences`도 인증 없이는 401이 되는지 확인하도록 추가했다.
- `scripts/smoke-production.mjs`
  - `PATCH /api/evidences`의 malformed JSON 처리를 스모크 검증에 추가했다.
- `README.md`, `src/lib/workspace-backup-export.ts`, `scripts/verify-workspace-backup-exports.ts`
  - 증빙 보관/삭제 기준을 메타데이터 수정과 개별 삭제 흐름으로 갱신했다.
