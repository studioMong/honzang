# 개발 진행 로그 - 2026-06-18 31

## 목표
- CSV import API가 반환하는 검증 오류를 업로드 화면에서 구체적으로 보여준다.
- 사용자가 “가져오기에 실패했습니다” 한 줄만 보고 원인을 추측하지 않게 한다.

## 변경 내용
- `src/components/app-workspace.tsx`
  - CSV 업로드 메시지 타입에 상세 오류 목록을 추가했다.
  - `/api/imports` 실패 응답의 `message`, `issues`를 화면 메시지로 변환한다.
  - 오류가 8개를 초과하면 남은 오류 수를 요약한다.
  - 기존 공용 `isRecord` 헬퍼를 재사용해 중복 정의를 피했다.
- `src/app/globals.css`
  - CSV 업로드 전용 `panel-message` 스타일을 추가했다.
  - 기존 `.import-message`를 쓰는 다른 폼에는 영향을 주지 않도록 별도 클래스로 분리했다.

## 검증
- `npm run typecheck` 통과.
- `npm run lint` 통과.

- 인앱 브라우저에서 `http://127.0.0.1:3000/?view=imports` 렌더링 확인.
