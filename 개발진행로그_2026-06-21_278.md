# 개발 진행 로그 278

## 시점

- 2026-06-21

## 진행 내용

- 운영자가 Railway 환경변수 설정에서 긴 랜덤값을 직접 만드는 실수를 줄이기 위해 시크릿 생성 도구를 추가했다.
- `npm run env:secrets`를 추가했다.
  - `NEXT_PUBLIC_APP_URL` 기본값은 `https://honzang-production.up.railway.app`.
  - `HONZANG_ACCESS_CODE`는 지정하지 않으면 랜덤 접근코드 초안을 생성한다.
  - `HONZANG_ACCESS_TOKEN_SALT`와 `HONZANG_FILE_ENCRYPTION_KEY`는 각각 32바이트 랜덤 base64url 값으로 생성한다.
  - `DATABASE_URL`은 출력하지 않고 Railway Postgres reference variable로 연결하도록 안내한다.
  - 생성값은 파일에 쓰지 않고 표준 출력으로만 표시한다.
- `verify:env-secrets`를 추가해 generator 출력 형식, 랜덤값 길이, `DATABASE_URL` 미출력, 문서 연결을 검증하게 했다.
- `verify:mvp`에 환경변수 시크릿 generator 검증 단계를 추가했다.
- README와 Railway cutover 체크리스트에 `npm run env:secrets` 사용법을 추가했다.
- 배포 설정 검증에 generator, verify script, README/cutover 문서 회귀 방지 assertion을 추가했다.

## 검증

- `node --check scripts/generate-env-secrets.mjs` 통과.
- `node --check scripts/verify-env-secret-generator.mjs` 통과.
- `npm run verify:env-secrets` 통과.
- `npm run verify:deployment-config` 통과.
- `npm run typecheck` 통과.
- `npm run lint` 통과.
- `npm run verify:mvp` 통과.

## 남은 조건

- 실제 운영 접근코드를 사용자가 확정하면 `npm run env:secrets` 출력 중 `HONZANG_ACCESS_CODE`를 원하는 값으로 대체하거나 아래처럼 생성한다.

```bash
HONZANG_ACCESS_CODE=운영_접근코드 npm run env:secrets
```

- 운영 접근코드가 제공되면 `verify:railway-authenticated`와 운영 DB mutation workflow를 순서대로 실행한다.
