# 개발진행로그 22

## 목표

CSV 없이도 누락 거래를 직접 입력하고, 대표자 입금이 매출로 잘못 분류되지 않게 한다.

## 변경 내용

- `/api/transactions`에 `POST`를 추가해 수기 거래를 생성할 수 있게 했다.
- 수기 거래 입력 시 거래일, 내용, 거래처, 입금/출금액, 확정 계정, 증빙 상태, 메모를 받을 수 있게 했다.
- 수기 거래는 입금과 출금 중 한쪽 금액만 입력되도록 검증한다.
- 거래 화면 상단에 `수기 거래` 입력 폼을 추가했다.
- sample mode에서도 수기 거래 생성 미리보기 응답을 반환해 DB 없이 화면 동작을 확인할 수 있게 했다.
- 대표자/개인/차입 관련 키워드가 일반 입금/매출 규칙보다 먼저 적용되도록 자동 분류 우선순위를 조정했다.
- 샘플 CSV 검증에 `대표자 입금 -> 대표자차입금(281)` 회귀 검증을 추가했다.

## 검증 결과

- `npm run verify:samples` 통과
- `npm run typecheck` 통과
- `npm run lint` 통과
- `npm run build` 통과
- 로컬 dev 서버에서 `/?view=transactions` HTTP 200 응답 확인
- 로컬 dev 서버에서 `POST /api/transactions` sample mode 응답 확인
- `대표자 입금` 수기 거래가 `대표자차입금(281)`으로 추천되는 것 확인
