# 개발진행로그 13

## 목표

웹과 앱을 모두 지원하겠다는 방향에 맞춰 설치형 PWA 기본 동작을 보강한다.

## 변경 내용

- manifest에 `id`, `scope`, `lang`, `orientation`, `categories`, 앱 바로가기 정보를 추가했다.
- production 환경에서 service worker를 등록하도록 연결했다.
- `public/sw.js`를 추가해 앱 쉘과 오프라인 페이지를 캐시한다.
- 장부 API 응답은 오래된 데이터 위험이 있으므로 service worker 캐시 대상에서 제외했다.
- `public/offline.html`을 추가해 네트워크 장애 시 기본 안내 화면을 제공한다.
- README에 설치형 PWA 기본 지원 범위를 기록했다.

## 검증 결과

- `npm run lint` 통과
- `npm run typecheck` 통과
- `npm run verify:samples` 통과
- `npm run build` 통과
- 로컬 서버에서 `/manifest.webmanifest`, `/sw.js`, `/offline.html` 모두 HTTP 200 응답 확인

## 제한 사항

- 네이티브 앱 래핑은 아직 진행하지 않았다. 우선 PWA로 웹/앱 공통 기반을 만든 뒤 필요 시 Capacitor, React Native, Flutter 중 하나를 선택한다.
