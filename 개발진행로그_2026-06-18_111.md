# 개발 진행 로그 111

## 작업 단위
- `PATCH /api/journals` 상태 변경 경로의 마감 잠금과 승인 중복 가드를 보강했다.

## 변경 파일
- `src/app/api/journals/route.ts`
- `scripts/verify-db-workflow.ts`

## 구현 내용
- 분개 상태 변경 시 분개일뿐 아니라 연결된 거래일도 마감 기간에 포함되는지 확인한다.
- 분개일은 열린 기간이더라도 연결 거래일이 마감된 기간이면 HTTP 409와 `PERIOD_CLOSED`를 반환한다.
- `DRAFT` 또는 `VOID` 분개를 `APPROVED`로 바꿀 때 같은 거래에 다른 승인 분개가 있으면 HTTP 409와 `APPROVED_JOURNAL_DUPLICATE_BLOCKED`를 반환한다.
- DB 워크플로우 검증에 6월 거래에 7월 분개일을 가진 승인 분개를 만든 뒤, 6월 마감 후 상태 변경이 차단되는 시나리오를 추가했다.

## 실행 명령
- `npm run typecheck`

- `git diff --check`
- `npm run verify:mvp`
