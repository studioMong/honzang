# 개발 진행 로그 - 2026-06-18 13

## 목표
- 최종 신고 점검에 빨간 차단 항목이 남아 있을 때 월 마감 잠금을 막는다.
- UI와 API 양쪽에서 같은 기준으로 마감을 차단해, 준비되지 않은 장부가 잠금 처리되는 흐름을 방지한다.

## 변경 내용
- `src/components/app-workspace.tsx`
  - 최종 신고 점검 결과에서 빨간 항목을 `closingBlockerRows`로 계산한다.
  - 차단 항목이 있으면 `마감 잠금` 버튼을 비활성화하고 상태 배지를 `마감 차단`으로 표시한다.
  - 차단 상태에서 마감 시도가 들어오면 어떤 점검 항목을 먼저 해결해야 하는지 안내한다.
- `src/app/api/closing-periods/route.ts`
  - 마감 저장 요청의 `summaryPayload` 안에 신고 점검 빨간 항목이 있으면 HTTP 409로 거절한다.
  - 응답 코드 `FILING_READINESS_BLOCKED`와 차단 항목 목록을 반환한다.
  - 기존 자동 검증용 legacy payload처럼 신고 점검 배열이 없는 입력은 기존 호환성을 위해 허용한다.
- `scripts/smoke-production.mjs`
  - 운영 smoke 테스트에 마감 차단 API 검증을 추가했다.
  - 빨간 신고 점검 항목을 보낸 경우 409와 `FILING_READINESS_BLOCKED`가 반환되는지 확인한다.
