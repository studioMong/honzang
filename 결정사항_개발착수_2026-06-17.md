# 혼자장부 개발 착수 결정사항

작성일: 2026-06-17

## 1. 사용자 결정사항

- 사업 형태: 내부 자체 재화형 결제와 SaaS 구독형을 모두 고려한다.
- 제품 형태: 웹과 앱을 모두 지원한다.
- 기술 기준: TypeScript 중심으로 진행한다.
- DB: Railway Postgres를 사용한다.
- 서버 주소: `honzang-production.up.railway.app`
- 대상: `혼자장부` 1인법인 사용자를 메인으로 한다.
- 사용자는 복식부기/세무 신고 흐름을 잘 모르는 상태이므로, 앱이 신고 준비에 필요한 리포트와 입력값을 쉽게 보여줘야 한다.

## 2. 기술 방향

초기 개발은 `Next.js + TypeScript + Prisma + PostgreSQL`로 진행한다.

이유:
- Railway 배포와 Postgres 연결이 단순하다.
- 웹앱과 API를 한 저장소에서 관리할 수 있다.
- TypeScript를 중심에 둘 수 있다.
- PWA로 먼저 만들면 모바일 브라우저 사용성이 확보된다.
- 이후 네이티브 앱이 필요하면 Capacitor로 같은 웹앱을 iOS/Android 앱으로 감싸는 방식이 Flutter보다 코드 재사용성이 높다.

Flutter는 다음 조건이 생길 때 검토한다.
- 오프라인 네이티브 저장소가 핵심이 되는 경우
- 카메라/문서 스캔/푸시/파일 접근 등 네이티브 기능이 제품 중심이 되는 경우
- 웹보다 모바일 앱 경험이 사업상 우선이 되는 경우

## 3. CSV 샘플 제공 방식

사용자는 실제 통장/카드/홈택스 CSV를 제공할 수 있다. 단, 지금은 어떤 파일을 줘야 하는지 명확하지 않으므로 앱에서 컬럼 매핑 방식을 지원한다.

추천 제공 방식:
- 법인통장 CSV: 실제 은행에서 내려받은 파일. 계좌번호와 상대방 이름은 마스킹해도 된다.
- 법인카드 CSV: 카드사 이용내역 다운로드 파일. 카드번호는 마스킹해도 된다.
- 홈택스 전자세금계산서 CSV: 매출/매입 구분이 보이는 내려받기 파일.
- 홈택스 현금영수증/카드 매입자료 CSV: 컬럼 구조가 실제와 같으면 금액/거래처는 일부 변경해도 된다.
- PG 정산 CSV: 네이버페이, 토스페이먼츠, 스마트스토어, 앱스토어 등 정산자료가 있으면 추가한다.

앱은 다음 흐름으로 처리한다.

1. CSV 업로드
2. 헤더 자동 인식
3. 사용자가 `거래일`, `적요`, `거래처`, `입금`, `출금`, `금액`, `부가세`, `승인번호` 등을 매핑
4. 매핑 템플릿 저장
5. 다음 업로드부터 자동 적용

## 4. 첫 버전 리포트 우선순위

복식부기를 모르는 사용자가 신고 준비에 바로 쓸 수 있어야 하므로 다음 리포트를 먼저 만든다.

1. 월별 손익 요약
   - 매출
   - 주요 비용
   - 예상 영업이익
   - 미분류/증빙누락 금액

2. 부가세 준비 리포트
   - 과세 매출 공급가액
   - 매출 부가세
   - 매입 공급가액
   - 매입 부가세
   - 불공제 후보
   - 예상 납부/환급액

3. 증빙/위험거래 검토 리포트
   - 증빙 없는 비용
   - 대표자 입출금
   - 법인카드 개인 사용 의심
   - 외주비 원천세 검토 필요
   - 해외 SaaS 결제

4. 신고 체크리스트
   - 이번 달 원천세
   - 이번 분기 부가세
   - 지급명세서/간이지급명세서
   - 법인세 결산 전 검토

5. 법인세 준비 패키지
   - 손익계산서 초안
   - 재무상태표 초안
   - 계정별 원장
   - 세무조정 검토 필요 항목

## 5. MVP 완성 기준

MVP는 다음 상태를 목표로 한다.

- 회사 1개를 설정할 수 있다.
- 통장/카드 CSV를 업로드하고 컬럼을 매핑할 수 있다.
- 거래 목록에서 계정과목을 지정할 수 있다.
- 자동 분류 규칙이 적용된다.
- 증빙 상태와 검토 필요 항목이 보인다.
- 월별 손익, 부가세 준비, 위험거래 리포트가 나온다.
- Railway Postgres에 데이터를 저장한다.
- Railway 배포를 위한 설정과 환경변수 예시가 준비된다.

## 6. 아직 사용자 확인이 필요한 항목

개발은 기본값으로 진행하되, 아래 정보가 오면 정확도를 높인다.

- 실제 `DATABASE_URL` 값은 공유하지 말고 Railway 환경변수에만 설정한다.
- 첫 실제 CSV 샘플 1개씩은 MVP 업로드 화면이 준비된 뒤 제공받는다.
- 실제 법인명/사업자번호는 개발 중에는 샘플값으로 두고, 배포 후 설정 화면에서 입력한다.
