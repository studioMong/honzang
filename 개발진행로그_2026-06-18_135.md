# 개발 진행 로그 135

## 작업 단위
- 리포트 저장/삭제가 기간 중간의 마감 잠금을 우회할 수 있는 문제를 보강했다.

## 변경 파일
- `src/lib/server/closing-periods.ts`
- `src/app/api/reports/route.ts`
- `scripts/verify-db-workflow.ts`

## 구현 내용
- `findClosedPeriodOverlappingRange` 헬퍼를 추가해 시작일과 종료일 사이에 겹치는 마감 기간을 찾도록 했다.
- `POST /api/reports`와 `DELETE /api/reports`가 시작일/종료일만 검사하지 않고 전체 기간 겹침으로 마감 잠금을 확인하도록 변경했다.
- 리포트 생성/삭제와 감사 로그 기록을 각각 하나의 Prisma 트랜잭션으로 묶었다.
- DB 워크플로우 검증에 연간 리포트가 6월 마감 잠금과 겹칠 때 생성/삭제가 모두 `PERIOD_CLOSED`로 차단되는 시나리오를 추가했다.
