# 개발 진행 로그 - 2026-06-18 15

## 목표
- CSV 컬럼 매핑 템플릿이 저장/적용되는 흐름을 사용자가 업로드 화면에서 확인할 수 있게 한다.
- 서버 응답과 클라이언트 상태를 맞춰, DB에 저장된 템플릿이 즉시 화면 상태에도 반영되게 한다.

## 변경 내용
- `src/lib/server/serializers.ts`
  - `CsvTemplate` 직렬화 함수를 추가했다.
  - Prisma JSON 매핑을 앱 타입의 `CsvColumnMapping`으로 반환하고 생성/수정 시각을 포함한다.
- `src/app/api/companies/route.ts`
  - 회사 조회 응답의 `csvTemplates`를 직렬화된 앱 객체로 반환한다.
- `src/app/api/imports/route.ts`
  - CSV 가져오기 성공 응답에 저장된 `csvTemplate`를 포함한다.
  - 중복 파일 업로드 응답에서도 방금 제출한 매핑을 최신 템플릿으로 저장하고 반환한다.
- `src/components/app-workspace.tsx`
  - 업로드 화면의 `가져오기 조건`에 `템플릿` 상태를 추가했다.
  - 파일 선택 전에는 저장 템플릿 개수 또는 `파일 선택 후` 상태를 보여준다.
  - 파일 선택 후 DB 템플릿, 브라우저 저장 매핑, 자동 추론, 직접 수정 상태를 구분한다.
  - 자료 유형을 바꾸면 현재 파일 헤더 기준으로 매핑을 다시 계산한다.
  - 가져오기 응답의 `csvTemplate`를 클라이언트 `csvTemplates` 상태에 즉시 병합한다.
- `scripts/verify-db-workflow.ts`
  - DB 연결 검증에서 CSV 가져오기 응답의 `csvTemplate`와 회사 조회의 템플릿 목록 반영 여부를 확인한다.
