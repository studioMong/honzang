# 개발 진행 로그 265

## 작업 시각

- 2026-06-18

## 목표

- 신고 근거 링크가 화면뿐 아니라 마감 스냅샷 export의 JSON, CSV, ZIP, XLSX에도 보존되는지 자동 검증한다.

## 변경 사항

- `scripts/verify-closing-snapshot-exports.ts`를 보강했다.
  - 기존 샘플 마감 스냅샷은 이전 데이터 호환을 위해 그대로 둔다.
  - 테스트 내부에서 `근거`, `근거 링크`가 포함된 신고 일정/제출 가이드 synthetic payload를 생성한다.
  - `closing-snapshot.json`에 국세청 URL이 보존되는지 확인한다.
  - `csv/filing-schedule.csv`에 `근거 링크` 컬럼과 URL이 보존되는지 확인한다.
  - `csv/submission-guide.csv`에 `국세청 부가세` 라벨과 URL이 보존되는지 확인한다.
  - ZIP 바이트에도 URL이 포함되는지 확인한다.
  - XLSX 바이트에는 XML escaping 때문에 `&`가 `&amp;`로 보존되는 것을 기준으로 확인한다.

## 검증

- `npm run verify:closing-snapshot-exports`
  - 최초 샌드박스 실행은 `tsx` IPC 제한으로 `listen EPERM` 실패.
  - 권한 상승 재실행 중 XLSX URL 원문 비교가 실패해 XML escaping 기대값으로 수정.
  - 수정 후 통과.
- `npm run typecheck`: 통과.
- `npm run lint`: 통과.
- `git diff --check`: 통과.
- `npm run verify:mvp`: 통과.
- `printenv DATABASE_URL`: 현재 셸에 값이 없어 `verify:db-workflow`는 보류.

## 메모

- 신고 근거 링크는 이제 화면 렌더링, CSV 다운로드, 신고 패키지 ZIP/JSON, 마감 스냅샷 XLSX까지 회귀 검증 대상이다.
