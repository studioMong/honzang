# 개발 진행 로그 45

## 목표

- 백업 복원 기능을 실제 DB 교체 없이 자동 검증할 수 있게 만든다.

## 작업 내용

- `scripts/verify-backup-restore.mjs`를 추가했다.
- 검증 스크립트는 production standalone 서버를 임시로 실행한다.
- 설정 화면 SSR에서 다음 문구를 확인한다.
  - `전체 백업`
  - `백업 JSON`
  - `백업 ZIP`
  - `백업 복원`
- `POST /api/backups/restore`에 `dryRun: true` 요청을 보내 백업 형식 검사와 복원 예정 개수를 확인한다.
- `confirmReplace` 없이 실제 복원 요청을 보내면 HTTP 400으로 거부되는지 확인한다.
- `package.json`에 `verify:backup-restore` 스크립트를 추가했다.
- README에 백업 복원 dry-run 검증 명령을 추가했다.

## 검증

- `npm run typecheck` 통과
- `npm run lint` 통과
- `npm run build` 통과
- `npm run verify:backup-restore` 통과
- `npm run verify:samples` 통과
- `npm run smoke:prod` 통과
- `npm run verify:pwa` 통과
