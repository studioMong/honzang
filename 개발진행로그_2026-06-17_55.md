# 개발진행로그 55 - CSV 자료별 준비 기준 표

## 진행 내용

- 업로드 화면에 `자료별 준비 기준` 표를 추가했다.
- 자료 유형별로 다음 정보를 한 화면에서 확인할 수 있게 했다.
  - 받을 곳
  - 필수 컬럼
  - 있으면 좋은 컬럼
  - 마스킹 가능 항목
  - 샘플 CSV 다운로드
- 통장, 카드, 홈택스 매출, 홈택스 매입, 현금영수증/카드 매입, PG 정산 자료 기준을 정리했다.
- 모바일에서는 표가 페이지 전체를 밀지 않고 표 영역 안에서만 가로 스크롤되도록 했다.

## 검증

- `npm run typecheck` 통과
- `npm run lint` 통과
- `npm run build` 통과
- `npm run verify:samples` 통과
- `npm run smoke:prod` 통과
- Browser 데스크톱 검증
  - `/?view=imports`에서 `자료별 준비 기준` 렌더링 확인
  - 6개 자료 유형 행 렌더링 확인
  - 통장 필수 컬럼과 마스킹 가능 항목 표시 확인
  - 페이지 전체 가로 오버플로우 없음
  - 콘솔 오류 없음
- Browser 모바일 폭 390px 검증
  - `자료별 준비 기준` 렌더링 확인
  - 표 영역 내부 가로 스크롤 구조 확인
  - 페이지 전체 가로 오버플로우 없음
  - 콘솔 오류 없음

## 참고

- 실제 CSV는 컬럼명이 달라도 업로드 후 매핑할 수 있다.
- 개발/검증용 샘플은 `public/samples/`에 계속 유지한다.
- 커밋 푸시 후 `RAILWAY_EXPECTED_COMMIT=9895f28 npm run verify:railway`를 실행했지만 공개 URL은 여전히 기존 정적 프로젝트 요약 페이지를 서빙하고 `/api/version`, `/api/health`, `/manifest.webmanifest`는 404로 응답했다.
