# 개발 진행 로그 145

## 작업 단위
- 검토 항목과 분개 상태 변경 API의 데이터 변경과 감사 로그 기록을 원자 작업으로 묶었다.

## 변경 파일
- `src/app/api/reviews/route.ts`
- `src/app/api/journals/route.ts`

## 구현 내용
- `PATCH /api/reviews`에서 검토 항목 상태 변경과 `REVIEW_STATUS_UPDATE` 감사 로그 기록을 하나의 Prisma 트랜잭션으로 처리한다.
- `PATCH /api/journals`에서 분개 상태 변경과 `JOURNAL_STATUS_UPDATE` 감사 로그 기록을 하나의 Prisma 트랜잭션으로 처리한다.
- 감사 로그 기록 실패 시 상태 변경만 남는 부분 저장 상태가 생기지 않도록 했다.

## 실행 명령
- `npm run typecheck`
- `npm run lint`
- `git diff --check`
- `npm run verify:mvp`
- `if [ -n "$DATABASE_URL" ]; then printf 'DATABASE_URL=set\n'; else printf 'DATABASE_URL=missing\n'; fi`
