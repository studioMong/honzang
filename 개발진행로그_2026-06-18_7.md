# 개발진행로그 2026-06-18 7

## 작업 범위
- 신고 준비 리포트에 `현금흐름 요약`을 추가했다.
- 기존 문서상 화면 구조에 있었지만 실제 앱에는 빠져 있던 현금흐름 리포트를 MVP 범위에 맞게 구현했다.

## 변경 내용
- `src/components/app-workspace.tsx`
  - 거래 CSV 기준 현금 유입/유출/순증감을 계산하는 `CashFlowRow`와 현금흐름 빌더를 추가했다.
  - 현금 유입을 `매출/영업 유입`, `대표자/자본 유입`, `기타 유입`으로 분류한다.
  - 현금 유출을 `영업 비용 유출`, `세금/공과 유출`, `대표자/자본 유출`, `기타 유출`로 분류한다.
  - 리포트 화면에 `현금흐름 요약` 섹션과 CSV 다운로드 버튼을 추가했다.
  - 최종 신고 점검, 홈택스 제출 전 입력 가이드, 법인세 결산 체크, 신고 패키지 요약에 현금흐름 항목을 연결했다.
  - 저장된 리포트 상세 화면에서도 현금흐름 요약을 다시 볼 수 있게 했다.
  - 신고 패키지 JSON, ZIP, XLSX에 `cash-flow.csv`와 `현금흐름` 시트를 포함했다.
- `scripts/smoke-production.mjs`
  - 리포트 화면 스모크 체크에 `현금흐름 요약` 문구를 추가했다.
- `README.md`
  - MVP 범위에 현금흐름 요약을 명시했다.

- `npm run typecheck` 통과.
- `npm run lint` 통과.
- `npm run build` 통과.
- `npm run verify:mvp` 통과.
- 브라우저 확인:
  - 데스크톱 1280px에서 `현금흐름 요약` 섹션 렌더링 확인.
  - 샘플 기준 현금 유입 `₩4,100,000`, 현금 유출 `₩482,000`, 순현금증감 `₩3,618,000`, 거래 `5건` 확인.
  - 모바일 390px에서 문서 가로 넘침 없음 확인.

## 판단
- 사용자가 복식부기를 몰라도 매출/대표자 입금/비용 지출이 실제 통장 흐름과 맞는지 대조할 수 있는 기본 리포트 축이 추가됐다.
- 향후 실제 통장 CSV의 시작/종료 잔액을 더 안정적으로 파싱하면 현금흐름 순증감과 잔액 차이를 자동 검증할 수 있다.
