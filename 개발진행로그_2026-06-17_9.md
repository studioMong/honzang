# 개발 진행 로그 - 2026-06-17 9차

## 이번 목표
- 저장된 신고 리포트 스냅샷을 다시 열어 세부 내용을 확인할 수 있게 만든다.
- 마감 당시 payload를 JSON으로 내보낼 수 있게 한다.

## 구현 내용
- `저장된 리포트` 표에 `열기` 작업을 추가했다.
- `리포트 상세` 섹션을 추가했다.
  - 기간, 거래 건수, 승인 분개 수, 매출 공급가액, 비용 공급가액, 손익, 예상 부가세, 증빙 누락, 원장 행 수를 표시한다.
  - 저장 당시 `신고 패키지` 항목을 표시한다.
  - 저장 당시 `법인세 체크` 항목을 표시한다.
  - 저장 당시 `원천세 후보` 항목을 표시한다.
- 저장된 리포트 JSON 다운로드 버튼을 추가했다.
- 저장 payload가 예상 구조와 다를 때도 화면이 깨지지 않도록 payload 정규화 헬퍼를 추가했다.

## 검증
- `npm run lint` 통과.
- `npm run typecheck` 통과.
- `npm run build` 통과.
- 로컬 브라우저 확인:
  - `/?view=reports`에서 `스냅샷 저장` 후 `저장된 리포트` 표시 확인.
  - `열기` 클릭 후 `리포트 상세` 섹션 표시 확인.
  - 상세 화면에서 `JSON`, `신고 패키지`, `법인세 체크`, `원천세 후보`, `매출 공급가액` 표시 확인.

## 다음 우선순위
- 상세 리포트에서 개별 표를 CSV로 다시 내보내는 기능을 추가할 수 있다.
- 저장된 스냅샷을 PDF/ZIP 패키지로 묶는 기능을 검토한다.
- Railway 대시보드/CLI 인증이 가능해지면 실제 Postgres 저장 스냅샷을 검증한다.
