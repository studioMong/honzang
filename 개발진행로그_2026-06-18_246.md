# 개발진행로그 2026-06-18 246

## 목표

- 검토함에서 열린 항목만 나열하는 수준을 넘어, 열린/완료/무시 상태를 빠르게 구분하고 처리 결과를 확인할 수 있게 한다.
- 검토 상태 변경도 접근제어, 마감 잠금, 감사 로그 검증 범위에 명시적으로 포함한다.

## 변경

- `src/components/app-workspace.tsx`
  - 검토함에 `열림`, `완료`, `무시`, `전체` 요약 필터 카드를 추가했다.
  - 기본 필터를 `열림`으로 두어 신고 전 처리할 항목이 먼저 보이게 했다.
  - 선택한 상태에 항목이 없으면 빈 상태 메시지를 분리해 보여준다.
  - 검토 상태 변경 중인 항목은 버튼을 비활성화하고 `처리 중` 상태를 보여준다.
  - 상태 변경 성공/실패 메시지를 검토함 상단에 표시한다.
  - 부모 `updateReviewStatus`가 성공/실패 결과를 반환하도록 바꿔 패널에서 오류를 처리할 수 있게 했다.
- `scripts/verify-access-control.mjs`
  - `PATCH /api/reviews`도 접근코드 없이 401이 되는지 확인하도록 추가했다.
- `scripts/verify-db-workflow.ts`
  - 마감 전 검토 항목 상태 변경 성공을 검증한다.
  - 마감 후 검토 상태 변경이 `PERIOD_CLOSED`로 차단되는지 기존 검증을 유지했다.
  - 감사 로그에 `REVIEW_STATUS_UPDATE`가 남는지 확인한다.
- `scripts/smoke-production.mjs`
  - 검토함 화면에 `열림`, `완료`, `무시`, `전체` 필터 텍스트가 노출되는지 확인한다.
- `README.md`
  - MVP 범위의 검토함 항목을 열린/완료/무시 필터가 있는 상태 관리 기능으로 갱신했다.

## 메모

- 검토함은 신고 전 실제 작업 대기열에 가까우므로 기본 화면은 `열림` 항목 중심으로 두었다.
- 완료/무시 항목은 사라지는 것이 아니라 필터로 다시 확인하거나 다시 열 수 있게 유지했다.
