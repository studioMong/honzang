# 개발진행로그 21

## 목표

잘못 업로드한 CSV 배치를 안전하게 되돌릴 수 있게 한다.

## 변경 내용

- `/api/imports`에 `DELETE`를 추가했다.
- 업로드 이력 행에 `삭제` 버튼을 추가했다.
- 삭제 성공 시 업로드 이력에서 해당 배치를 제거하고 전체 데이터를 refresh한다.
- 승인된 분개가 연결된 거래가 있으면 삭제를 막고 안내 메시지를 반환한다.
- 삭제 가능한 배치는 연결 증빙을 거래에서 분리하고, 검토 항목, 분개 초안, 거래, import batch를 정리한다.

## 안전 조건

- `APPROVED` 분개가 있는 import batch는 삭제하지 않는다.
- `DRAFT` 분개는 거래와 함께 삭제한다.
- 증빙 파일/증빙 기록 자체는 삭제하지 않고 거래 연결만 해제한다.
