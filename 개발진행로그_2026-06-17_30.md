## 판단

- 현재 저장소에는 루트 정적 `index.html`이 없고, `railway.json`은 Dockerfile 빌드를 지정하고 있다.
- 로컬 production build와 standalone smoke는 통과했다.
- 따라서 코드 문제보다는 Railway 서비스가 최신 GitHub 커밋을 빌드하지 않거나, 공개 도메인이 다른 서비스/이전 배포에 연결된 상태일 가능성이 높다.

## Railway 대시보드에서 확인할 항목

- 서비스가 `studioMong/honzang` 저장소의 `main` 브랜치에 연결되어 있는지
- 최신 배포 커밋이 `d2a60b3` 이상인지
- Build 로그에서 `Dockerfile` 빌드가 감지되는지
- Deploy 로그에서 `npm run db:deploy`, `npm run start`가 실행되는지
- Variables에 `DATABASE_URL`, `NEXT_PUBLIC_APP_URL=https://honzang-production.up.railway.app`가 설정되어 있는지
- 공개 도메인 `honzang-production.up.railway.app`이 현재 Next 앱 서비스에 연결되어 있는지
