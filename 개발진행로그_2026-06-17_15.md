# 개발진행로그 15

## 목표

사용자가 같은 CSV 파일을 반복 업로드해 거래가 중복 생성되는 문제를 방지한다.

## 변경 내용

- import API에서 자료 유형, 헤더, 행 데이터를 canonical JSON으로 정규화한 뒤 SHA-256 해시를 계산한다.
- DB 연결 모드에서는 같은 회사, 같은 자료 유형, 같은 파일 해시의 import batch가 있으면 새 거래를 만들지 않고 기존 거래를 반환한다.
- `ImportBatch.originalFileHash` 복합 인덱스를 Prisma schema와 migration에 추가했다.
- 업로드 화면에 신규 가져오기/중복 파일 메시지를 표시한다.
- `ImportPayload` 타입에 `headers`와 `originalFileHash`를 반영했다.
